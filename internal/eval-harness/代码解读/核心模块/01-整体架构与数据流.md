## 核心数据结构

### AgentTrajectory — 轨迹（最重要的数据结构）

```typescript
// 定义于 src/matchers.ts:22
interface AgentTrajectory {
  steps: AgentStep[];              // 代理的每一轮行为
  files: Record<string, string>;   // 执行结束后的文件系统快照
}

// 定义于 src/matchers.ts:8
interface AgentStep {
  index: number;           // 步骤编号，从 1 开始
  action: AIMessage;       // 模型的输出（可能包含工具调用）
  observations: ToolMessage[];  // 工具执行的结果
}
```

**一个典型的轨迹长这样：**

```
AgentTrajectory {
  steps: [
    step 1: {
      action: AIMessage("我来帮你创建文件", tool_calls: [{ name: "write", args: {...} }])
      observations: [ToolMessage("文件已创建")]
    },
    step 2: {
      action: AIMessage("文件已创建完成，内容如下...")
      observations: []          // 最后一步通常无工具调用
    }
  ],
  files: {
    "src/hello.ts": "export function hello() { return 'hello'; }"
  }
}
```

### EvalRunner — 运行器接口

```typescript
// 定义于 src/runners.ts:49
interface EvalRunner {
  name: string;
  run(params: RunAgentParams): Promise<AgentTrajectory>;
  extend(overrides: Record<string, unknown>): EvalRunner;
}

interface RunAgentParams {
  query: string;
  initialFiles?: Record<string, string>;  // 预置文件
}
```

`extend()` 的设计是关键：它让你在不修改已注册运行器的情况下，临时覆盖配置（如系统提示、工具列表）：

```typescript
const customRunner = runner.extend({
  systemPrompt: "你是一个专注于 TypeScript 的代码助手"
});
```

## 消息流转过程

LangGraph 执行代理后，返回的 `messages` 数组结构如下：

```
index 0:  HumanMessage("用户的查询")        ← parseTrajectory 跳过此项
index 1:  AIMessage(tool_calls: [...])      ← Step 1 的 action
index 2:  ToolMessage("工具结果")           ← Step 1 的 observation
index 3:  AIMessage(tool_calls: [...])      ← Step 2 的 action
index 4:  ToolMessage("工具结果")           ← Step 2 的 observation
index 5:  AIMessage("最终回复文本")          ← Step 3 的 action（无工具调用）
```

`parseTrajectory()` 的分组规则：
- 每遇到 `AIMessage`，开启一个新的 step
- 随后的 `ToolMessage` 归属于当前 step 的 `observations`
- 时间复杂度 O(n)，单次线性扫描

## 文件格式转换

LangGraph 内部使用 `FileData` 格式存储文件（按行分割）：

```typescript
// LangGraph 内部格式（FileData）
{
  "src/main.ts": {
    content: ["line1", "line2", "line3"],
    created_at: "2024-01-01T00:00:00Z",
    modified_at: "2024-01-01T00:00:00Z"
  }
}

// coerceFiles() 转换后（供测试断言使用）
{
  "src/main.ts": "line1\nline2\nline3"
}
```

**双向转换：**
- **输入**（`initialFiles`）：字符串 → FileData（在 `DeepAgentEvalRunner.run()` 中）
- **输出**（`result.files`）：FileData → 字符串（在 `coerceFiles()` 中）

## 运行器注册机制

```
运行时初始化（vitest setupFile）
  ↓
runners.ts 中调用 7 次 registerDeepAgentRunner()
  ↓
全局 runners 字典：{
  "sonnet-4-5":         DeepAgentEvalRunner,
  "sonnet-4-5-thinking": DeepAgentEvalRunner,
  "opus-4-6":           DeepAgentEvalRunner,
  "sonnet-4-6":         DeepAgentEvalRunner,
  "gpt-4.1":            DeepAgentEvalRunner,
  "gpt-4.1-mini":       DeepAgentEvalRunner,
  "o3-mini":            DeepAgentEvalRunner,
}
  ↓
测试中调用 getDefaultRunner()
  ↓
读取 process.env.EVAL_RUNNER（如 "sonnet-4-5"）
  ↓
返回对应的 EvalRunner 实例（结果被缓存）
```
