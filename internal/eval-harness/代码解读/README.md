# eval-harness 源码解读

> **目标**：帮助你理解 `@deepagents/evals` 的设计思路，以便构建类似的 AI 代理评估框架。

## 项目定位

`@deepagents/evals` 是一个专为 AI 代理（Agent）系统设计的**评估测试框架**。它解决了一个核心问题：

> 如何在自动化测试中验证"对话式 AI 代理"的行为是否符合预期？

普通的单元测试断言（`expect(result).toBe("hello")`）无法应对 AI 代理的输出特点：
- 输出是多轮交互的序列（调用工具 → 观察结果 → 再调用工具）
- 文件系统状态在执行过程中发生变化
- 需要对比不同模型（Claude、GPT）的行为差异

该框架通过以下三件事解决这些问题：
1. **统一运行接口**：用一个通用接口抽象不同的底层代理实现
2. **轨迹结构化**：把代理执行过程中的消息流解析为可断言的结构
3. **专用断言器**：为 vitest 扩展针对代理轨迹的 matcher

## 关键术语速查

| 术语               | 说明                                                                       |
| ------------------ | -------------------------------------------------------------------------- |
| `AgentTrajectory`  | 代理一次完整运行的记录：步骤列表 + 最终文件快照                            |
| `AgentStep`        | 轨迹中的一个步骤：模型输出（`AIMessage`）+ 工具调用结果（`ToolMessage[]`） |
| `EvalRunner`       | 抽象接口，封装"运行代理并返回轨迹"的能力                                   |
| `RunAgentParams`   | 运行代理的输入：用户查询 + 可选的初始文件                                  |
| `parseTrajectory`  | 核心函数：把 LangGraph 的消息数组解析为结构化 `AgentTrajectory`            |
| `registerRunner`   | 把一个 `EvalRunner` 实例注册到全局注册表                                   |
| `getDefaultRunner` | 读取 `EVAL_RUNNER` 环境变量，返回对应的运行器实例                          |
| `FileData`         | LangGraph 内部的文件格式：`{ content: string[], created_at, modified_at }` |
| LangSmith          | 实验追踪平台，每次断言自动上报评估指标                                     |
