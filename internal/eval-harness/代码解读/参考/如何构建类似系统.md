# 如何构建类似的 AI 代理评估系统

本文从 `@deepagents/evals` 的设计中提炼出关键模式，帮助你构建自己的 AI 代理评估框架。

---

## 核心设计模式

### 1. 统一运行器接口（Adapter Pattern）

这是整个框架最重要的设计决策。无论底层是什么代理（LangGraph、AutoGen、自研框架），测试代码只和 `EvalRunner` 接口打交道：

```typescript
interface EvalRunner {
  name: string;
  run(params: RunAgentParams): Promise<AgentTrajectory>;
  extend(overrides: Record<string, unknown>): EvalRunner;
}
```

**你需要实现的：**

```typescript
class YourAgentRunner implements EvalRunner {
  name = "your-agent-v1";

  async run(params: RunAgentParams): Promise<AgentTrajectory> {
    // 1. 把 params.query 转换为你的代理的输入格式
    // 2. 执行代理（可能是 HTTP 请求、本地调用、gRPC 等）
    // 3. 把代理输出转换为 AgentTrajectory
    return {
      steps: [...],
      files: {...},
    };
  }

  extend(overrides: Record<string, unknown>): EvalRunner {
    // 返回配置变体，不修改原实例
    return new YourAgentRunner({ ...this.config, ...overrides });
  }
}
```

**关键点：**
- `run()` 必须是无状态的（每次调用都是独立的）
- `extend()` 必须返回新实例，不修改 `this`
- 把底层格式转换为标准 `AgentTrajectory` 是适配器的核心工作

### 2. 注册表 + 环境变量 驱动测试

```typescript
// 注册
const runners: Record<string, EvalRunner> = {};
function registerRunner(runner: EvalRunner) {
  runners[runner.name] = runner;
}

// 解析
function getDefaultRunner(): EvalRunner {
  const name = process.env.EVAL_RUNNER;
  return runners[name];
}
```

**为什么这个模式好：**
- 同一份测试代码，换一个环境变量就能测试不同模型
- CI 可以矩阵化测试：`EVAL_RUNNER=model-a vitest && EVAL_RUNNER=model-b vitest`
- 无需修改代码就能切换"被测系统"

---

### 3. 轨迹（Trajectory）作为测试的核心对象

不要只看代理的"最终输出"，要关注整个执行过程：

```typescript
interface AgentTrajectory {
  steps: AgentStep[];              // 每一步做了什么
  files: Record<string, string>;   // 文件系统最终状态
}

interface AgentStep {
  index: number;
  action: AIMessage;               // 模型的决策
  observations: ToolMessage[];     // 工具的反馈
}
```

**轨迹让你能验证：**
- 代理是否经历了正确的步骤数（效率）
- 代理是否调用了正确的工具（行为路径）
- 代理是否以正确的参数调用工具（精确性）
- 代理最终给出了正确的回答（正确性）
- 文件系统最终状态是否符合预期（副作用）

### 4. 自定义断言器（Custom Matchers）

为你的轨迹类型扩展测试框架的断言能力：

```typescript
// vitest 扩展示例
expect.extend({
  toHaveUsedTool(trajectory: AgentTrajectory, toolName: string) {
    const used = trajectory.steps
      .flatMap(s => s.action.tool_calls ?? [])
      .some(tc => tc.name === toolName);
    return {
      pass: used,
      message: () => `expected agent to use tool "${toolName}"`,
    };
  }
});

// jest 扩展示例
expect.extend({
  toHaveUsedTool(received, toolName) {
    // 同样的逻辑
  }
});
```

### 5. 量化指标与人工评估分离

`@deepagents/evals` 在每次断言时自动上报 LangSmith 指标。如果你没有 LangSmith，可以替换为自己的日志系统：

```typescript
// 替换 ls.logFeedback 为自己的指标上报
function logMetric(key: string, value: number, metadata: Record<string, unknown>) {
  // 发送到 Prometheus、DataDog、CSV、SQLite...
  metrics.record({ key, value, ...metadata, timestamp: Date.now() });
}

// 在 matcher 中调用
toHaveAgentSteps(received, expected) {
  const actual = received.steps.length;
  logMetric("agent_steps", actual, { expected, model: process.env.EVAL_RUNNER });
  return { pass: actual === expected, ... };
}
```

## 构建最小可行评估系统的步骤

### 第一步：定义轨迹数据结构

根据你的代理特点定义适合的轨迹结构。如果你的代理没有"工具调用"概念，可以简化：

```typescript
interface MinimalTrajectory {
  turns: Array<{
    userInput?: string;
    agentOutput: string;
    metadata?: Record<string, unknown>;
  }>;
  finalOutput: string;
}
```

### 第二步：实现轨迹解析

把代理框架的原始输出转换为你的轨迹格式：

```typescript
function parseYourAgentOutput(rawOutput: YourAgentOutput): MinimalTrajectory {
  return {
    turns: rawOutput.history.map((turn, i) => ({
      userInput: turn.user,
      agentOutput: turn.assistant,
    })),
    finalOutput: rawOutput.finalResponse,
  };
}
```

### 第三步：实现 EvalRunner

```typescript
class YourEvalRunner implements EvalRunner {
  constructor(
    public name: string,
    private config: YourAgentConfig,
  ) {}

  async run(params: RunAgentParams): Promise<MinimalTrajectory> {
    const rawOutput = await callYourAgent(params.query, this.config);
    return parseYourAgentOutput(rawOutput);
  }

  extend(overrides: Partial<YourAgentConfig>): EvalRunner {
    return new YourEvalRunner(this.name, { ...this.config, ...overrides });
  }
}
```

### 第四步：注册运行器

```typescript
// setup.ts（在测试初始化时运行）
registerRunner(new YourEvalRunner("model-a", { model: "gpt-4" }));
registerRunner(new YourEvalRunner("model-b", { model: "claude-3" }));
registerRunner(new YourEvalRunner("model-a-verbose", {
  model: "gpt-4",
  verboseMode: true,
}));
```

### 第五步：编写测试

```typescript
// eval.test.ts
it("completes task in reasonable steps", async () => {
  const runner = getRunner(process.env.EVAL_RUNNER ?? "model-a");
  const traj = await runner.run({ query: "帮我写一个排序函数" });
  
  // 使用内置断言
  expect(traj.finalOutput).toContain("function");
  
  // 使用自定义 matcher（如果实现了的话）
  expect(traj).toHaveCompletedSuccessfully();
  expect(traj).toHaveTurnsLessThan(5);
});
```

## 常见扩展点

### 扩展 1：添加新模型

只需在 `setup.ts` 中添加一行注册：

```typescript
registerRunner(new YourEvalRunner("claude-4", {
  model: "claude-opus-4-6",
  apiKey: process.env.ANTHROPIC_API_KEY,
}));
```

### 扩展 2：添加"黄金测试"（Golden Test）

```typescript
expect.extend({
  toMatchGoldenTrajectory(received: AgentTrajectory, goldenFile: string) {
    const golden = JSON.parse(fs.readFileSync(goldenFile, "utf-8"));
    const pass = deepEqual(received.steps.map(s => s.action.tool_calls), golden);
    return { pass, message: () => "trajectory doesn't match golden" };
  }
});
```

### 扩展 3：并行评估

```typescript
async function runBenchmark(query: string) {
  const runners = ["model-a", "model-b", "model-c"].map(getRunner);
  const results = await Promise.all(
    runners.map(r => r.run({ query }).then(traj => ({ runner: r.name, traj })))
  );
  return results;
}
```

### 扩展 4：评估套件管理

```typescript
// 将多个测试用例定义为数据，与断言逻辑分离
const testCases = [

testCases.forEach(({ id, query, expectedSteps, expectedTools }) => {
```

## 本框架的局限与你可能需要解决的问题

| 局限 | 说明 | 可能的解决方案 |
|------|------|-------------|
| 单轮对话 | 每次 `run()` 是独立会话，不支持多轮交互评估 | 扩展 `RunAgentParams`，添加 `conversationHistory` 字段 |
| 同步执行 | `invoke()` 等待完整输出，无法观察流式过程 | 添加 streaming 模式，记录 token 级别的延迟 |
| 无执行超时 | 代理可能无限循环 | 在 `run()` 中加 `Promise.race([agent.invoke(), timeout(30000)])` |
| 无重试机制 | 网络错误会直接失败 | 封装带指数退避的重试逻辑 |
| JSON.stringify 参数比较 | `argsEquals` 对键顺序敏感 | 改用深度相等比较（如 `deep-equal` 包） |
