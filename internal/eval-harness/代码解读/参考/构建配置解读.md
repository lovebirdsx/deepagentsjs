# 构建配置解读

## tsdown.config.ts

```typescript
import { defineConfig } from "tsdown";

const external = [/^[^./]/];  // 所有不以 . 或 / 开头的模块视为外部依赖

export default defineConfig([
  {
    entry: ["./src/index.ts"],
    format: ["esm"],
    dts: true,        // 生成 .d.ts 类型定义文件
    clean: true,      // 构建前清空输出目录
    sourcemap: true,  // 生成 source map
    outDir: "dist",
    outExtensions: () => ({ js: ".js" }),  // ESM 产物扩展名：.js
    external,
  },
  {
    entry: ["./src/index.ts"],
    format: ["cjs"],
    dts: true,
    clean: true,
    sourcemap: true,
    outDir: "dist",
    outExtensions: () => ({ js: ".cjs" }),  // CJS 产物扩展名：.cjs
    external,
  },
]);
```

### 双格式输出的原因

同时输出 ESM（`.js`）和 CJS（`.cjs`）是 npm 包的最佳实践，确保兼容性：

| 使用方式 | 加载的文件 |
|---------|----------|
| `import { ... } from "@deepagents/evals"` | `dist/index.js`（ESM） |
| `const { ... } = require("@deepagents/evals")` | `dist/index.cjs`（CJS） |
| TypeScript ESM 类型 | `dist/index.d.ts` |
| TypeScript CJS 类型 | `dist/index.d.cts` |

### `external` 正则表达式

```typescript
const external = [/^[^./]/];
// 匹配所有不以 . 或 / 开头的字符串
// 即：所有 npm 包名（如 "langsmith"、"@langchain/core"）
// 不匹配：相对路径（"./matchers.js"）、绝对路径（"/usr/..."）
```

这确保所有 npm 依赖不被打包进产物，而是保留为运行时依赖。

## package.json 关键配置

### exports 字段（条件导出）

```json
"exports": {
  ".": {
    "import": {
      "types": "./dist/index.d.ts",
      "default": "./dist/index.js"
    },
    "require": {
      "types": "./dist/index.d.cts",
      "default": "./dist/index.cjs"
    }
  },
  "./setup": {
    "import": {
      "types": "./dist/index.d.ts",
      "default": "./dist/index.js"
    }
  },
  "./package.json": "./package.json"
}
```

`"."` 路径（默认导出）：用于 `import from "@deepagents/evals"`  
`"./setup"` 路径：用于 vitest 的 `setupFiles: ["@deepagents/evals/setup"]`  
`"./package.json"`：允许外部读取包的元数据

### 版本与发布状态

```json
"version": "0.0.7",
"private": true
```

`"private": true` 表示这个包不会发布到 npm registry，只在 monorepo 内部使用。版本号 `0.0.7` 表明仍处于早期迭代阶段。

### workspace 依赖

```json
"dependencies": {
  "deepagents": "workspace:*"
}
```

`workspace:*` 是 pnpm workspace 的语法，表示引用同一 monorepo 中的 `deepagents` 包（`libs/deepagents`），不从 npm 拉取。

## tsconfig.json

```json
{
  "extends": "@tsconfig/recommended",
  "compilerOptions": {
    "outDir": "dist",
    "noUnusedLocals": false,
    "noUnusedParameters": false,
    "types": ["node"]
  },
  "include": ["src/**/*"]
}
```

- `extends "@tsconfig/recommended"`：使用推荐的严格模式配置基线
- `noUnusedLocals/Parameters: false`：关闭未使用变量警告（评估代码中可能有临时变量）
- `types: ["node"]`：包含 Node.js 内置类型（`process.env` 等）
