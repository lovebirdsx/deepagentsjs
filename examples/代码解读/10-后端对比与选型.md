# 后端对比与选型

**文件**：`backends/` 目录下的 5 个文件

后端（Backend）是 deepagents 文件系统的存储层抽象。`backends/` 目录通过 5 个独立示例展示了 5 种后端，每个示例都是一个使用相同研究工作流但不同存储策略的 Agent。

---

## 5 种后端一览

| 后端 | 持久化 | 跨线程 | 可执行 Shell | 适用场景 |
|------|--------|--------|------------|---------|
| `StateBackend` | ❌（会话内） | ❌ | ❌ | 默认，无需配置 |
| `StoreBackend` | ✅（数据库） | ✅ | ❌ | 跨对话持久化文件 |
| `FilesystemBackend` | ✅（磁盘） | ✅ | ❌ | 操作真实文件 |
| `CompositeBackend` | 混合 | 混合 | ❌ | 按路径前缀分路由 |
| `LocalShellBackend` | ✅（磁盘） | ✅ | ✅ | 文件 + 命令执行 |

---

## StateBackend（默认）

**文件**：`backends/state-backend.ts`

```typescript
export const agent = createDeepAgent({
  tools: [internetSearch],
  systemPrompt: `Your files are stored in memory and will be LOST when conversation ends.
    ## Workflow
    1. Write research question to research_question.txt
    2. Gather information using internet_search
    3. Write findings to research_notes.txt
    4. Write summary to summary.md`,
  backend: new StateBackend(),   // 显式指定，实际上这也是默认值
});
```

**行为**：
- 文件存储在 LangGraph 的 State（`AgentState.files` 字段）
- 每次 `invoke()` 调用后随 State 一起销毁（除非使用 checkpointer 持久化 State）
- 并发子代理的文件操作通过 `fileDataReducer` 合并（无竞态）

**何时用**：
- 快速原型，不需要文件持久化
- Agent 只是用文件做中间存储，最终答案在消息里

---

## StoreBackend（跨线程持久化）

**文件**：`backends/store-backend.ts`

```typescript
export const agent = createDeepAgent({
  tools: [internetSearch],
  systemPrompt: `All files persist across ALL conversations.
    You can reference previous research in new conversations.`,
  checkpointer: new MemorySaver(),   // 必须有 checkpointer
  store: new InMemoryStore(),         // 必须有 store
  backend: new StoreBackend(),
});

// 第一次对话：研究 AI 趋势，写入文件
await agent.invoke(
  { messages: [new HumanMessage("Research AI trends for 2025")] },
  { configurable: { thread_id: "thread-1" } },
);

// 第二次对话（不同 thread）：文件仍然可访问
await agent.invoke(
  { messages: [new HumanMessage("Do you have info on AI trends?")] },
  { configurable: { thread_id: "thread-2" } },   // 不同 thread！
);
```

**行为**：
- 文件存储在 LangGraph `BaseStore`（生产中可用 PostgreSQL 等）
- **跨线程共享**：不同 `thread_id` 的对话看到相同文件
- 需要同时配置 `checkpointer` 和 `store`

**何时用**：
- 需要积累知识库（研究报告集合、代码库分析等）
- 多个对话协作使用同一份文件
- 类似"共享知识库"的场景

## FilesystemBackend（磁盘文件）

**文件**：`backends/filesystem-backend.ts`

```typescript
const workspaceDir = path.join(process.cwd(), "workspace");

export const agent = createDeepAgent({
  systemPrompt: `You are an expert coding assistant with access to the real filesystem.
    Files persist after the conversation ends.`,
  backend: new FilesystemBackend({
    rootDir: workspaceDir,
    virtualMode: true,   // 路径视为相对于 rootDir 的虚拟路径
  }),
});
```

**行为**：
- 文件直接读写磁盘（`fs.writeFileSync`/`fs.readFileSync`）
- 对话结束后文件仍然保留
- `virtualMode: true`：Agent 只能看到 `rootDir` 内部（安全隔离）

**何时用**：
- 编码助手（需要修改真实代码文件）
- ACP Server 集成（IDE 文件直接操作）
- 需要与其他程序共享文件

## CompositeBackend（按路径分路由）

**文件**：`backends/composite-backend.ts`

```typescript
export const agent = createDeepAgent({
  systemPrompt: `
    ## Storage Types
    1. Temporary files (root directory): Stored in state, lost after conversation
       - /research_notes.txt, /draft.md
    2. Persistent memory (/memories/ directory): Stored in database, kept forever
       - /memories/report_2025_ai_trends.md
    
    ## Workflow
    5. Save final report to /memories/report_TOPIC.md (persistent)`,
  checkpointer: new MemorySaver(),
  store: new InMemoryStore(),
  backend: new CompositeBackend(
    new StateBackend(),          // 默认后端（处理根目录文件）
    {
      "/memories/": new StoreBackend(),  // 路径前缀路由到 Store
    }
  ),
});
```

**行为**：路径前缀决定使用哪个后端：
- `/memories/` 开头 → `StoreBackend`（持久化）
- 其他路径 → `StateBackend`（临时）

**何时用**：
- 需要区分临时工作区和持久化存储
- 分层存储策略（热数据 + 冷数据）
- 这是最灵活的方案，可以组合任意多个后端

## LocalShellBackend（文件 + Shell 执行）

**文件**：`backends/local-shell-backend.ts`

```typescript
export const agent = createDeepAgent({
  systemPrompt: `You have filesystem tools AND shell execution.
    Execute: install dependencies, run tests, build projects.`,
  backend: new LocalShellBackend({
    rootDir: workspaceDir,
    virtualMode: true,
    inheritEnv: true,    // 继承当前进程的环境变量
    timeout: 60,         // Shell 命令超时（秒）
  }),
});

// 任务：创建 Node.js 项目，安装依赖，写并运行测试
await agent.invoke({
  messages: [new HumanMessage(
    "Create a simple Node.js project with a package.json, " +
    "install vitest, and write a basic test that passes.",
  )],
});
```

**行为**：
- 继承 `FilesystemBackend` 的所有文件操作
- 额外提供 `execute` 工具，执行任意 Shell 命令
- `inheritEnv`：继承父进程环境变量（npm、node 等 PATH）

**何时用**：
- 本地开发助手（需要执行 `npm install`、`npm test` 等）
- 代码生成 + 验证（写完代码立即运行）
- 不需要云隔离的场景

## 选型决策树

```
需要执行代码/Shell 命令？
    ├─ 是 → LocalShellBackend（本地）或沙箱后端（隔离）
    └─ 否 → 继续
        │
        文件需要跨对话保留？
            ├─ 是 → 需要读写真实文件？
            │       ├─ 是 → FilesystemBackend
            │       └─ 否 → StoreBackend
            └─ 否 → 需要按路径分不同存储策略？
                    ├─ 是 → CompositeBackend
                    └─ 否 → StateBackend（默认）
```
