# 流式输出完全指南

**文件**：`streaming/` 目录下的 8 个文件

DeepAgents 基于 LangGraph 的流式 API，支持多种流式模式。8 个示例覆盖了从基础到高级的完整用法。

---

## LangGraph 流式基础

```typescript
// 非流式：等待全部完成
const result = await agent.invoke({ messages: [...] });

// 流式：边执行边接收
for await (const [namespace, chunk] of await agent.stream(
  { messages: [...] },
  {
    streamMode: "updates",  // 流式模式
    subgraphs: true,        // 包含子代理事件
  },
)) {
  // namespace: string[]  — 事件来源（空数组=主 Agent，有值=子代理）
  // chunk: any           — 事件数据（根据 streamMode 格式不同）
}
```

**关键参数**：
- `streamMode`：决定接收什么类型的数据
- `subgraphs: true`：开启子代理事件传播

---

## 流式模式（streamMode）

| 模式 | 接收内容 | 格式 | 适用场景 |
|------|---------|------|---------|
| `"updates"` | 每个节点的状态变化 | `{ nodeName: data }` | 进度追踪、调试 |
| `"messages"` | LLM token 流 | `[BaseMessage, metadata]` | 实时显示文字 |
| `"custom"` | 工具发送的自定义事件 | 任意 JSON | 自定义进度条 |
| `["updates", "messages", "custom"]` | 以上全部 | `[ns, mode, data]` | 完整监控 |

## 示例 1：基础子图流（basic.ts）

```typescript
for await (const [namespace, chunk] of await agent.stream(
  { messages: [{ role: "user", content: "Research quantum computing" }] },
  { streamMode: "updates", subgraphs: true },
)) {
  if (namespace.length > 0) {
    console.log(`[subagent: ${namespace.join("|")}]`);  // 子代理事件
  } else {
    console.log("[main agent]");  // 主 Agent 事件
  }
  console.log(chunk);
}
```

**要点**：通过 `namespace` 区分主 Agent 和子代理事件。
- `namespace = []`：主 Agent
- `namespace = ["tools:uuid"]`：子代理（uuid 是子代理的 pregel 任务 ID）

## 示例 2：逐 token 流（tokens.ts）

```typescript
for await (const [namespace, chunk] of await agent.stream(
  { messages: [...] },
  { streamMode: "messages", subgraphs: true },
)) {
  const [message] = chunk;           // BaseMessage 对象
  const isSubagent = namespace.some(s => s.startsWith("tools:"));

  if (message.text) {
    process.stdout.write(message.text);  // 逐 token 写入终端
  }
}
```

`streamMode: "messages"` 每次 yield 一个 `[BaseMessage, metadata]` 元组。`message.text` 是当前 token 片段（可能是几个字符）。

## 示例 3：工具调用流（tool-calls.ts）

```typescript
import { AIMessageChunk, ToolMessage } from "langchain";

for await (const [namespace, chunk] of await agent.stream(...)) {
  const [message] = chunk;

  // 工具调用开始（流式 args）
  if (AIMessageChunk.isInstance(message) && message.tool_call_chunks?.length) {
    for (const tc of message.tool_call_chunks) {
      if (tc.name) console.log(`\nTool call: ${tc.name}`);
      if (tc.args) process.stdout.write(tc.args);  // args 分块流入
    }
  }

  // 工具调用结果
  if (ToolMessage.isInstance(message)) {
    console.log(`Tool result [${message.name}]: ${message.text?.slice(0, 150)}`);
  }

  // 普通文字 token
  if (AIMessageChunk.isInstance(message) && message.text && !message.tool_call_chunks?.length) {
    process.stdout.write(message.text);
  }
}
```

**要点**：工具调用的 `args` 也是流式传输的，需要累积才能得到完整 JSON。

## 示例 4：步骤进度追踪（progress.ts）

```typescript
for await (const [namespace, chunk] of await agent.stream(
  ...,
  { streamMode: "updates", subgraphs: true },
)) {
  if (namespace.length === 0) {
    // 主 Agent 步骤
    for (const [nodeName, data] of Object.entries(chunk)) {
      if (nodeName === "tools") {
        // 子代理返回结果
        for (const msg of data.messages ?? []) {
          if (msg.type === "tool") {
            console.log(`Subagent complete: ${msg.name}`);
            console.log(`Result: ${String(msg.content).slice(0, 200)}...`);
          }
        }
      } else {
        console.log(`[main agent] step: ${nodeName}`);
      }
    }
  } else {
    // 子代理步骤
    for (const [nodeName] of Object.entries(chunk)) {
      console.log(`  [${namespace[0]}] step: ${nodeName}`);
    }
  }
}
```

**输出示例**（来自代码注释）：
```
[main agent] step: patchToolCallsMiddleware.before_agent
[main agent] step: SummarizationMiddleware.before_model
[main agent] step: model_request
  [tools:3a048147] step: model_request
  [tools:3a048147] step: tools
Subagent complete: task
  Result: ...
```

这揭示了 deepagents 中间件的真实执行顺序（每个中间件对应一个节点）。

## 示例 5：子代理生命周期（lifecycle.ts）

```typescript
const activeSubagents = new Map<string, { type, description, status }>();

for await (const [namespace, chunk] of await agent.stream(...)) {
  for (const [nodeName, data] of Object.entries(chunk)) {

    // Phase 1: 检测子代理启动（主 Agent 模型请求中有 task 工具调用）
    if (namespace.length === 0 && nodeName === "model_request") {
      for (const msg of data.messages ?? []) {
        for (const tc of msg.tool_calls ?? []) {
          if (tc.name === "task") {
            activeSubagents.set(tc.id, { type: tc.args.subagent_type, status: "pending" });
            console.log(`PENDING → subagent "${tc.args.subagent_type}" (${tc.id})`);
          }
        }
      }
    }

    // Phase 2: 子代理开始执行（收到 tools:uuid namespace 事件）
    if (namespace.length > 0 && namespace[0].startsWith("tools:")) {
      // 标记为 running
    }

    // Phase 3: 子代理完成（主 Agent tools 节点返回 tool message）
    if (namespace.length === 0 && nodeName === "tools") {
      for (const msg of data.messages ?? []) {
        if (msg.type === "tool") {
          activeSubagents.get(msg.tool_call_id).status = "complete";
          console.log(`COMPLETE → ${msg.tool_call_id}`);
        }
      }
    }
  }
}
```

**注意**：tool_call_id（LLM 生成）和 pregel task ID（LangGraph 分配）是**不同的 ID**，需要区分。

## 示例 6：按来源分流（filter-by-type.ts）

```typescript
const buffers = new Map<string, string>();  // 按来源积累文字

for await (const [namespace, chunk] of await agent.stream(
  ...,
  { streamMode: "messages", subgraphs: true },
)) {
  const [message] = chunk;
  const text = message.text ?? extractFromContentBlocks(message.content);
  if (!text) continue;

  const isSubagent = namespace.some(s => s.startsWith("tools:"));
  const key = isSubagent
    ? namespace.find(s => s.startsWith("tools:"))
    : "main";

  buffers.set(key, (buffers.get(key) ?? "") + text);
}

// 按来源打印完整文字
for (const [key, text] of buffers) {
  console.log(`─── ${key === "main" ? "main agent" : `subagent (${key})`} ───`);
  console.log(text.trim());
}
```

## 示例 7：自定义进度事件（custom-updates.ts）

```typescript
// 工具内部发送进度事件
const analyzeData = tool(
  async ({ topic }, config: ToolRuntime) => {
    const writer = config.writer;  // 自定义事件发送器

    writer?.({ status: "starting", topic, progress: 0 });
    await new Promise(r => setTimeout(r, 500));

    writer?.({ status: "analyzing", progress: 50 });
    await new Promise(r => setTimeout(r, 500));

    writer?.({ status: "complete", progress: 100 });
    return `Analysis result...`;
  },
  { name: "analyze_data", schema: z.object({ topic: z.string() }) },
);

// 接收自定义事件
for await (const [namespace, chunk] of await agent.stream(
  ...,
  { streamMode: "custom", subgraphs: true },
)) {
  const isSubagent = namespace.some(s => s.startsWith("tools:"));
  console.log(`[${isSubagent ? "subagent" : "main"}]`, chunk);
  // chunk: { status: "starting", topic: "...", progress: 0 }
}
```

**关键**：`config.writer` 是 LangGraph 提供的通道，只在 `streamMode: "custom"` 或多模式时可接收。

```typescript
)) {
  // 多模式时：chunk 是 3 元组 [namespace, mode, data]
  if (mode === "updates") {
    // 步骤进度
  } else if (mode === "messages") {
    // token 流
  } else if (mode === "custom") {
    // 自定义事件
  }
}
```

**注意**：多模式时 yield 的格式变为 `[namespace, mode, data]`（3 元组），单模式是 `[namespace, data]`（2 元组）。

## 流式模式选型建议

| 需求 | 推荐模式 |
|------|---------|
| 显示文字生成过程 | `"messages"` |
| 显示步骤进度（调试/监控） | `"updates"` |
| 工具进度条（自定义事件） | `"custom"` |
| 子代理生命周期追踪 | `"updates"` |
| 全功能监控 | `["updates", "messages", "custom"]` |
| 只关注主 Agent 输出 | `"messages"` + 过滤 namespace |
