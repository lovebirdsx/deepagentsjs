# 异步子代理并行研究

**文件**：
- `async-subagents/parallel-research/supervisor.ts` — 督导 Agent
- `async-subagents/parallel-research/researcher.ts` — 研究者子图（异步）
- `async-subagents/parallel-research/ui/` — React 前端 UI

这是最复杂的示例之一，演示**多个研究者在后台并行执行**，并通过 Completion Callback 通知督导 Agent 完成情况，配备 React UI 展示任务状态。

---

## 架构图

```
用户（浏览器 UI）
    │ 发送消息
    ▼
supervisor 图（LangGraph Platform）
    │ start_async_task × N
    ▼
researcher 图 × N（各自独立执行）
    │ 完成时通过 completion callback 通知
    ▼
supervisor 收到通知，告知用户结果可查
```

运行环境：两个图都部署在 **LangGraph Platform**（本地用 `langgraph-cli dev` 启动）。

## supervisor.ts 解读

### AsyncSubAgent 声明

```typescript
const asyncSubAgents: AsyncSubAgent[] = [
  {
    name: "researcher",
    description: "A general-purpose research agent that can investigate any topic.",
    graphId: "researcher",   // 对应 langgraph.json 中注册的图 ID
    // 不指定 url：使用 LangGraph Platform 默认地址
  },
];

export const graph = createDeepAgent({
  systemPrompt: "...",
  subagents: asyncSubAgents,
});
```

`AsyncSubAgent` 没有 `tools` 和 `systemPrompt` 字段（那些在 researcher 图自己内部定义），只有：
- `name`：在主 Agent 中调用时的名字
- `description`：LLM 用来决定何时启动它
- `graphId`：LangGraph Platform 上的图 ID
- `url`（可选）：远程服务地址

### 督导 Agent 的提示策略

督导 Agent 的系统提示明确划分了操作模式：

```
// 只有用户明确要求研究时才启动（不要轻易扇出）
Only launch researchers when user uses: "research", "investigate", "deep dive", "look into"

// 启动任务时的流程
1. 决定启动几个研究者，每个关注什么角度
2. 用 start_async_task 启动，agentName: "researcher"
3. 告知用户哪些研究者在工作

// 查询结果时
Never report task statuses from memory — always call list_async_tasks for live statuses

// 取消任务时
Use cancel_async_task with the exact task_id
```

**设计要点**：用详细的规则约束 LLM 的行为，避免频繁错误启动后台任务。

## researcher.ts 解读

### 网络搜索工具（有无 API Key 的两路实现）

```typescript
let searchImpl: (query: string) => Promise<string>;

if (process.env.TAVILY_API_KEY) {
  // 真实 Tavily 搜索
  searchImpl = async (query) => {
    const res = await fetch("https://api.tavily.com/search", { ... });
    return formatResults(data.results);
  };
} else {
  // Stub 搜索（开发时不需要 API Key）
  searchImpl = async (query) => {
    return `1. Key finding: Recent developments in ${query}...`;
  };
}
```

这种双路实现让示例无需依赖外部 API 也能运行，便于开发测试。

### Completion Callback 中间件

```typescript
const notifier = createCompletionCallbackMiddleware({
  callbackGraphId: "supervisor",   // 完成后通知 supervisor 图
  url: LANGGRAPH_URL,              // LangGraph Platform 地址
});

export const graph = createDeepAgent({
  tools: [webSearch],
  middleware: [notifier],   // 添加回调中间件
});
```

`createCompletionCallbackMiddleware` 的工作方式：
1. 研究者图完成时，自动向 supervisor 图发送一个消息
2. 消息内容包含 task_id 和任务结果
3. supervisor 收到消息后知道哪个研究者完成了

## React UI 解读

`ui/src/App.tsx` 是一个用 `@langchain/langgraph-sdk` 流式接口驱动的聊天界面：

```typescript
// 关键：使用 LangGraph SDK 流式通信
import { Client } from "@langchain/langgraph-sdk";

const client = new Client({ apiUrl: "http://localhost:2024" });

// 流式接收 supervisor 的消息和任务状态更新
for await (const chunk of client.runs.stream(threadId, "supervisor", {
  input: { messages: [userMsg] },
  streamMode: ["messages", "updates"],
})) {
  // 根据 chunk.event 类型处理：
  // - "messages/partial" → 追加消息文本
  // - "updates" → 更新任务状态卡片
}
```

`ui/src/components/ResearcherCard.tsx` 展示每个异步研究者的状态（pending/running/complete）和结果。

## 异步任务的五个操作

deepagents 为 AsyncSubAgent 提供了五个内置工具：

| 工具 | 对应操作 | 说明 |
|------|---------|------|
| `start_async_task` | 启动 | 创建后台研究任务，返回 task_id |
| `check_async_task` | 查询 | 获取单个任务的状态和结果 |
| `list_async_tasks` | 列举 | 获取所有任务的当前状态 |
| `update_async_task` | 更新 | 给正在运行的任务发新指令 |
| `cancel_async_task` | 取消 | 终止一个任务 |
