# 技能文件解读

**文件**：`skills/` 目录下的 4 个技能和 1 个技能创建工具

SKILL.md 是 deepagents 技能系统的核心，每个技能是一个目录，内含 `SKILL.md`（必须）和可选的可执行脚本/资源文件。

---

## SKILL.md 格式规范

```markdown
---
name: skill-name          # 必须，kebab-case，≤64字符
description: ...          # 必须，LLM 用来决定是否使用，≤1024字符
compatibility: "TS+Node"  # 可选，运行环境说明
license: MIT              # 可选
metadata:                 # 可选，任意扩展字段
  version: "1.0.0"
  author: "Acme Corp"
---

# 技能标题

## When to Use
- 触发场景描述

## How to Use / Workflow
- 具体操作步骤

## Best Practices
- 注意事项
```

## 4 个技能分析

### 1. arxiv-search — arXiv 论文搜索

**目录**：`skills/arxiv-search/`

```markdown
---
name: arxiv-search
description: Search arXiv preprint repository for papers in physics, 
             mathematics, computer science, quantitative biology, and related fields
---
```

**配套脚本**：`arxiv_search.ts`（通过 `execute` 工具运行）

```bash
# Agent 使用方式
npx tsx /path/to/skills/arxiv-search/arxiv_search.ts "deep learning" --max-papers 5
```

**技能设计特点**：
- `description` 列举了所有支持的学科领域（让 LLM 知道何时用）
- 提供 `execute` 命令行接口而非直接 API 调用（灵活，任何语言可实现）
- 输出格式固定（Title + Summary），便于 Agent 解析

**脚本工作流**：
1. Agent 调用 `execute("npx tsx arxiv_search.ts 'query'")`
2. 脚本调用 arXiv API（无需 API Key）
3. 返回格式化的论文列表

### 2. web-research — 结构化网络研究

**目录**：`skills/web-research/`

这是一个**流程型技能**，不提供工具，而是提供详细的工作流程指导：

```markdown
---
name: web-research
description: Use this skill for web research requests; provides structured approach 
             to conducting comprehensive web research
---

## Research Process

### Step 1: Create and Save Research Plan
1. Create research folder: mkdir research_[topic]
2. Analyze research question into distinct subtopics
3. Write research_plan.md before delegating

### Step 2: Delegate to Research Subagents
- Use task tool with 3-5 max searches per subtopic
- Run up to 3 subagents in parallel

### Step 3: Synthesize Findings
- list_files research_[topic] → 查看子代理产出
- read_file research_[topic]/findings_*.md → 读取内容
- 综合成最终报告
```

**设计特点**：
- 没有配套脚本，纯粹是操作流程指导
- 明确规定了并行上限（≤3 个子代理）和搜索上限（≤5 次/子代理）
- 强调"先计划，再委派"（写 research_plan.md）
- 区分了 `read_file`（本地文件）和 `fetch_url`（网络 URL）

### 3. skill-creator — 技能创建指南（元技能）

**目录**：`skills/skill-creator/`

这是一个**元技能**——用来帮助创建其他技能的技能：

```markdown
---
name: skill-creator
description: "Guide for creating effective skills... Use this skill when the user asks to: 
              create a skill, make a skill, build a skill, set up a skill, initialize a skill, 
              scaffold a skill, update or modify an existing skill, validate a skill..."
---

## Skill Creation Process
1. Understand the skill with concrete examples
2. Plan reusable skill contents (scripts, references, assets)
3. Initialize the skill (run init_skill.ts)
4. Edit the skill
5. Validate the skill (run quick_validate.ts)
6. Iterate based on real usage
```

**配套脚本**：
- `scripts/init_skill.ts`：初始化技能目录脚手架（创建标准目录结构和 SKILL.md 模板）
- `scripts/quick_validate.ts`：验证技能格式（检查 YAML frontmatter、命名规范等）

**description 设计亮点**：非常详细地列举了触发词（create/make/build/set up/initialize/scaffold/update/modify/validate），让 LLM 在各种表达方式下都能识别需要使用此技能。

### 4. file-organizer、langgraph-docs（无详细注解）

这两个技能文件未在代码中体现用法，但遵循相同格式：

## 技能目录标准结构

```
skill-name/
├── SKILL.md                    # 必须：元数据 + 操作指南
├── scripts/                    # 可选：可执行脚本
│   ├── main_script.ts
│   └── helper.ts
├── references/                 # 可选：参考文档（按需加载到上下文）
│   └── api-reference.md
└── assets/                     # 可选：模板/图标等静态资源
    └── report-template.md
```

## 技能设计原则（来自 skill-creator 的指导）

### 精简原则
> The context window is a public good. Only add context the agent doesn't already have.

- 只描述 Agent **不知道**的内容（通用知识不需要）
- 使用示例比长段解释更有效
- 触发条件（When to Use）要精确，避免误触发

### description 字段的重要性

description 是技能在**渐进式加载**中唯一始终可见的字段，决定 LLM 是否使用此技能：

```
// 差的 description
description: "A skill for research"  ← 太模糊

// 好的 description
description: "Use this skill for requests related to web research; it provides 
              a structured approach to conducting comprehensive web research"
```

好的 description 应该：
1. 包含典型触发词
2. 说明技能提供什么（流程？工具？知识？）
3. 控制在 1024 字符内

## 技能与 AGENTS.md 的分工

| | Skills（SKILL.md） | Memory（AGENTS.md） |
|--|--------------------|--------------------|
| 内容类型 | 操作流程、工具用法 | 项目上下文、代码规范 |
| 加载方式 | 按需（LLM 决定） | 全量（启动时） |
| 更新场景 | 添加新能力 | 更新项目状态 |
| 大小限制 | 单文件 10MB | 无限制（合理控制）|
