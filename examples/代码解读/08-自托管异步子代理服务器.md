# 自托管异步子代理服务器

**文件**：
- `async-subagent-server/server.ts` — Agent Protocol 服务端（Hono + PostgreSQL）
- `async-subagent-server/supervisor.ts` — 对接服务器的督导 Agent REPL

演示如何**不依赖 LangGraph Platform**，自己用 Hono + PostgreSQL 搭建一个符合 Agent Protocol 的服务器，供 DeepAgents 的 AsyncSubAgent 对接。

---

## 架构图

```
supervisor（本地 REPL）
    │ HTTP 请求（Agent Protocol）
    ▼
server（Hono + PostgreSQL）
    ├── POST /threads               → 创建会话
    ├── POST /threads/:id/runs      → 创建任务（fire-and-forget）
    ├── GET  /threads/:id/runs/:id  → 查询状态（轮询）
    ├── GET  /threads/:id/state     → 获取结果
    └── POST /threads/:id/runs/:id/cancel → 取消任务
        │
        ▼ 异步执行
    createDeepAgent（研究员）
```

## server.ts 解读

### 数据库设计

```sql
-- threads 表：对话会话，存储消息历史和最终输出
CREATE TABLE IF NOT EXISTS threads (
  thread_id  TEXT        PRIMARY KEY,
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  messages   JSONB       NOT NULL DEFAULT '[]',  -- { role, content }[] 数组
  output     TEXT                                 -- 最终 AI 输出（NULL 直到成功）
);

-- runs 表：一次运行尝试
CREATE TABLE IF NOT EXISTS runs (
  run_id       TEXT        PRIMARY KEY,
  thread_id    TEXT        NOT NULL REFERENCES threads(thread_id),
  assistant_id TEXT        NOT NULL,
  status       TEXT        NOT NULL DEFAULT 'pending',  -- pending|running|success|error|cancelled
  created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  error        TEXT
);
```

**设计说明**：
- 一个 thread 可以有多个 run（重试、更新指令）
- output 字段是 thread 级别的，被最后一次成功的 run 写入
- 使用 JSONB 存储消息数组（简单方案，生产环境可考虑更严格的类型）

### Agent Protocol 端点实现

#### POST /threads — 创建会话

```typescript
app.post("/threads", async (c) => {
  const threadId = uuidv4();
  const { rows } = await pool.query<Thread>(
    `INSERT INTO threads (thread_id) VALUES ($1)
     RETURNING thread_id, created_at, messages, output`,
    [threadId],
  );
  return c.json(rows[0]);
});
```

每次 `start_async_task` 都会先调用此接口创建一个新 thread。

#### POST /threads/:threadId/runs — 创建运行

```typescript
app.post("/threads/:threadId/runs", async (c) => {
  const body = await c.req.json();

  // 处理 update_async_task（interrupt 策略）
  if (body.multitask_strategy === "interrupt") {
    // 取消所有正在运行的 runs
    await pool.query(
      `UPDATE runs SET status = 'cancelled' WHERE thread_id = $1 AND status = 'running'`,
      [thread.thread_id],
    );
    // 清空当前输出（新任务从头开始）
    await pool.query("UPDATE threads SET output = NULL WHERE thread_id = $1", [...]);
  }

  // 创建新 run 记录
  const run = await pool.query<Run>(
    `INSERT INTO runs (run_id, thread_id, assistant_id) VALUES ($1, $2, $3) RETURNING *`,
    [runId, threadId, assistantId],
  );

  // 关键：fire-and-forget，不等待 Agent 完成
  executeRun(run.run_id, run.thread_id, userMessage);  // 不 await！

  return c.json(run.rows[0]);   // 立即返回，客户端去轮询
});
```

**设计亮点**：`executeRun` 不被 `await`，服务器立即返回 run 对象，客户端通过 `check_async_task` 轮询状态。

#### GET /threads/:threadId/state — 获取结果

```typescript
app.get("/threads/:threadId/state", async (c) => {
  const thread = await getThread(c.req.param("threadId"));
  return c.json({
    values: {
      messages: thread.messages,  // Agent Protocol 标准格式
    },
    next: [],
    metadata: {},
  });
});
```

deepagents 的 `check_async_task` 在 run 变为 `success` 后，读取 `values.messages` 最后一条消息作为结果。

### Agent 执行函数

```typescript
async function executeRun(runId: string, threadId: string, input: string): Promise<void> {
  // 1. 标记为运行中
  await pool.query("UPDATE runs SET status = 'running' WHERE run_id = $1", [runId]);
  
  try {
    // 2. 调用 Agent
    const result = await agent.invoke({
      messages: [new HumanMessage(input)],
    });
    
    // 3. 提取最后一条 AI 消息
    const output = typeof lastMessage.content === "string"
      ? lastMessage.content
      : JSON.stringify(lastMessage.content);
    
    // 4. 写入数据库
    await pool.query(
      `UPDATE threads SET output = $1, messages = messages || $2::jsonb WHERE thread_id = $3`,
      [output, JSON.stringify([{ role: "assistant", content: output }]), threadId],
    );
    
    // 5. 标记成功
    await pool.query("UPDATE runs SET status = 'success' WHERE run_id = $1", [runId]);
  } catch (e) {
    await pool.query(
      "UPDATE runs SET status = 'error', error = $1 WHERE run_id = $2",
      [String(e), runId],
    );
  }
}
```

## supervisor.ts 解读

### REPL 结构

```typescript
// 指定明确的远程服务地址（与 parallel-research 不同，那个用 Platform）
const asyncSubAgents: AsyncSubAgent[] = [
  {
    name: "researcher",
    description: "...",
    graphId: "researcher",
    url: RESEARCHER_URL,   // "http://localhost:2024"
  },
];

// 使用 MemorySaver 保持对话状态（跨轮次记住 task_id）
const checkpointer = new MemorySaver();
const threadId = uuidv4();

const supervisor = createDeepAgent({
  checkpointer,
  systemPrompt: "...",
  subagents: asyncSubAgents,
});

// 带状态的对话
async function chat(userInput: string) {
  const result = await supervisor.invoke(
    { messages: [new HumanMessage(userInput)] },
    { configurable: { thread_id: threadId } },   // 通过 thread_id 关联历史
  );
}
```

**关键**：`MemorySaver` + `thread_id` 让用户在多轮对话中可以用 task_id 引用之前启动的任务。

### 系统提示的行为规范

```
START: 用户说 "research X"
  → 调用 start_async_task，返回 taskId

CHECK: 用户说 "check status of <id>"
  → 调用 check_async_task，报告结果

UPDATE: 用户说 "change task <id> to focus on Y"
  → 调用 update_async_task，发新指令

CANCEL: 用户说 "cancel <id>"
  → 调用 cancel_async_task

LIST: 用户说 "list all tasks"
  → 调用 list_async_tasks

Rules:
- 永远不要从记忆中报告状态，必须调用工具获取实时状态
- 每次用户请求只调用一个工具
- 永远显示完整 taskId
```

## 注意：取消的局限性

```typescript
// 取消只是更新数据库状态，不中断 Agent 进程
app.post("/threads/:threadId/runs/:runId/cancel", async (c) => {
  await pool.query("UPDATE runs SET status = 'cancelled' WHERE run_id = $1", [run.run_id]);
  // ↑ Agent 的 executeRun 仍在继续执行，只是结果不会被采用
});
```

注释中明确说明：真正的中断需要 `AbortController` 或作业队列。这是一个示例实现的已知简化。
