# 沙箱执行环境

**文件**：`sandbox/` 目录下的 6 个文件

沙箱后端在文件系统后端的基础上增加了 **`execute` 工具**，让 Agent 能在隔离环境中执行 Shell 命令。6 个示例覆盖了从云端到本地的各种隔离级别。

---

## 6 种沙箱对比

| 沙箱 | 包 | 隔离级别 | 网络 | 前置条件 |
|------|----|---------|------|---------|
| DaytonaSandbox | `@langchain/daytona` | 云端容器 | ✅ | Daytona API Key |
| DenoSandbox | `@langchain/deno` | 云端 Linux microVM | ✅ | Deno Deploy Token |
| ModalSandbox | `@langchain/modal` | 云端容器 | ✅ | Modal Token |
| VfsSandbox | `@langchain/node-vfs` | 内存虚拟 FS | ❌ | 无 |
| LangSmithSandbox | `deepagents`（内置） | 云端容器 | ✅ | LangSmith API Key |
| 自定义 BaseSandbox | `deepagents` | 本地 Shell | 本地 | 无 |

---

## DaytonaSandbox — 云端容器

**文件**：`sandbox/daytona-sandbox.ts`

```typescript
// 1. 创建沙箱（分配云端资源）
const sandbox = await DaytonaSandbox.create({
  language: "typescript",
  autoStopInterval: 15,   // 15 分钟无活动自动停止
  labels: { purpose: "example" },
});

try {
  // 2. 使用沙箱作为后端
  const agent = createDeepAgent({
    model: new ChatAnthropic({ model: "claude-haiku-4-5" }),
    systemPrompt: "...",
    backend: sandbox,   // 沙箱直接作为 backend 传入
  });

  await agent.invoke({
    messages: [new HumanMessage(
      "Create hello.ts, run it with npx tsx, show output."
    )],
  });
} finally {
  // 3. 必须清理沙箱（避免费用）
  await sandbox.close();
}
```

**核心**：`DaytonaSandbox.create()` 是异步的（分配云端资源），必须用 try/finally 确保 `close()`。

---

## DenoSandbox — Linux microVM

**文件**：`sandbox/deno-sandbox.ts`

```typescript
const sandbox = await DenoSandbox.create({
  memoryMb: 1024,       // 1GB 内存
  lifetime: "session",  // 脚本结束时自动销毁
});

// 用法与 Daytona 相同
const agent = createDeepAgent({ backend: sandbox, ... });
```

DenoSandbox 跑在 Deno Deploy 的 Linux microVM 里，比容器更轻量，启动更快。

---

## ModalSandbox — 容器镜像可定制

**文件**：`sandbox/modal-sandbox.ts`

```typescript
const sandbox = await ModalSandbox.create({
  imageName: "python:3.12-slim",  // 指定基础镜像（Python 环境）
  timeoutMs: 600_000,              // 10 分钟超时
});
```

Modal 支持自定义 Docker 镜像，适合需要特定运行环境（如 Python ML 环境、特定版本 Node.js）的场景。

## VfsSandbox — 内存虚拟文件系统

**文件**：`sandbox/vfs-sandbox.ts`

```typescript
const sandbox = await VfsSandbox.create({
  initialFiles: {
    // 预置文件
    "/package.json": JSON.stringify({ name: "vfs-demo", version: "1.0.0" }),
    "/README.md": "# VFS Demo",
  },
});

try {
  const agent = createDeepAgent({ backend: sandbox, ... });
  // Agent 在内存虚拟 FS 中操作，不影响真实磁盘
} finally {
  await sandbox.stop();   // stop 而不是 close
}
```

**特点**：
- 无需网络、无需 API Key、纯内存
- 可以预置初始文件
- 结束后所有文件自动消失
- 基于 Node.js VFS polyfill（等待官方 `node:vfs` 落地）

## LangSmithSandbox — 内置云沙箱

**文件**：`sandbox/langsmith-sandbox.ts`

```typescript
// 内置于 deepagents 包，不需要额外安装
import { createDeepAgent, LangSmithSandbox } from "deepagents";

const backend = await LangSmithSandbox.create({
  templateName: "deepagents",   // LangSmith 中预创建的沙箱模板
});

const agent = createDeepAgent({ backend, ... });
```

LangSmith 沙箱直接内置在 `deepagents` 包中，不需要安装额外的 provider 包。

## 自定义 BaseSandbox — 本地 Shell

**文件**：`sandbox/local-sandbox.ts`

这是最有参考价值的示例，展示了**如何自己实现一个沙箱**：

```typescript
export class LocalShellSandbox extends BaseSandbox {
  readonly id: string;
  private readonly workingDirectory: string;
  private readonly timeout: number;

  constructor(options: { workingDirectory: string; timeout?: number }) {
    super();   // BaseSandbox 自动实现 ls/read/write/glob/grep（基于 execute）
    this.workingDirectory = path.resolve(options.workingDirectory);
    this.timeout = options.timeout ?? 30000;
    this.id = `local-shell-${...}`;
    fs.mkdirSync(this.workingDirectory, { recursive: true });
  }

  // 只需实现 3 个方法
  async execute(command: string): Promise<ExecuteResponse> {
    return new Promise((resolve) => {
      const child = cp.spawn("/bin/bash", ["-c", command], {
        cwd: this.workingDirectory,
        env: { ...process.env, HOME: process.env.HOME },
      });
      
      // 收集 stdout + stderr
      child.stdout.on("data", collectOutput);
      child.stderr.on("data", collectOutput);
      
      // 超时处理
      const timer = setTimeout(() => {
        child.kill("SIGTERM");
        resolve({ output: chunks.join("") + "\n[Command timed out]", exitCode: null });
      }, this.timeout);
      
      child.on("close", (exitCode) => {
        clearTimeout(timer);
        resolve({ output: chunks.join(""), exitCode, truncated });
      });
    });
  }

  async uploadFiles(files: Array<[string, Uint8Array]>): Promise<FileUploadResponse[]> {
    // 将文件写入 workingDirectory
  }

  async downloadFiles(paths: string[]): Promise<FileDownloadResponse[]> {
    // 从 workingDirectory 读取文件
  }
}
```

**继承 BaseSandbox 的好处**：
- 只需实现 `execute`、`uploadFiles`、`downloadFiles` 三个方法
- `BaseSandbox` 自动基于这三个方法实现 `ls`、`read`、`write`、`glob`、`grep` 等所有后端接口
- 相当于用 Shell 命令实现了整个 `BackendProtocol`

## 统一的系统提示模板

所有沙箱示例共享相同的系统提示结构：

```
## Tools Available
- execute: Run any shell command
- ls: List directory contents
- read_file: Read file contents
- write_file: Create new files
- edit_file: Modify existing files
- grep: Search for patterns
- glob: Find files

## Best Practices
1. Start by exploring: ls or execute("ls -la")
2. Use execute for complex commands
3. Chain: execute("npm install && npm test")
4. Check exit codes to verify success
```

## 选型建议

```
需要隔离代码执行？
    ├─ 不需要网络访问 → VfsSandbox（最简单，无需配置）
    ├─ 需要网络，有 LangSmith → LangSmithSandbox
    ├─ 需要定制镜像（Python/其他语言）→ ModalSandbox
    ├─ 想用 Deno 生态 → DenoSandbox
    ├─ 想用 Daytona → DaytonaSandbox
    └─ 本地开发，不需要云 → LocalShellBackend（backends/）
                             或自定义 BaseSandbox（更好的控制）
```
