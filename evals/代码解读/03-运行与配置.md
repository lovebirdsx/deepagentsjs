# 运行与配置

## 环境变量

| 变量 | 必需 | 说明 |
|------|------|------|
| `EVAL_RUNNER` | **必需** | 选择运行的模型，如 `sonnet-4-6`、`gpt-4.1`、`o3-mini` |
| `ANTHROPIC_API_KEY` | 使用 Claude 时 | Anthropic API 密钥 |
| `OPENAI_API_KEY` | 使用 GPT/o3 时 | OpenAI API 密钥 |
| `LANGSMITH_API_KEY` | 可选但推荐 | LangSmith 实验追踪 |
| `LANGSMITH_PROJECT` | 可选 | LangSmith 项目名（默认使用套件名） |
| `OOLONG_MAX_PER_DATASET` | 可选 | oolong 套件每个数据集的最大用例数（默认 10） |
| `OOLONG_CONTEXT_LEN` | 可选 | oolong 套件过滤特定上下文长度 |

## 运行命令

### 运行单个套件

```bash
# 进入套件目录
cd evals/basic

# 设置环境变量并运行
EVAL_RUNNER=sonnet-4-6 LANGSMITH_API_KEY=... pnpm test:eval
```

### 运行所有套件

```bash
cd evals/all
EVAL_RUNNER=sonnet-4-6 pnpm test:eval
```

### 不发送 LangSmith 数据运行（本地调试）

```bash
EVAL_RUNNER=sonnet-4-6 vitest run
```

## 套件 package.json 结构

每个套件都是独立的 pnpm workspace 包：

```json
{
  "name": "@deepagents/evals-basic",
  "version": "0.0.0",
  "private": true,
  "scripts": {
    "test:eval": "vitest run"
  },
  "dependencies": {
    "@deepagents/evals": "workspace:*"
  }
}
```

## vitest.config.ts 结构

每个套件的 Vitest 配置基本一致：

```typescript
import { defineConfig } from "vitest/config";

export default defineConfig({
  test: {
    // 注册自定义断言器 + 7 个模型 Runner
    setupFiles: ["@deepagents/evals/setup"],

    // 使用 LangSmith reporter 上报测试结果
    reporters: ["default", "langsmith/vitest/reporter"],

    // 关闭并发（避免 API 限流）
    pool: "forks",
    poolOptions: { forks: { singleFork: true } },

    // 较长超时（真实 LLM 调用）
    testTimeout: 300_000,  // 5 分钟
  },
});
```

## 套件文件结构（标准模板）

```
evals/{suite-name}/
├── package.json          ← 包配置，依赖 @deepagents/evals
├── vitest.config.ts      ← Vitest 配置（基本相同）
├── eval.test.ts          ← 测试入口，导入 index.ts 的套件函数并调用
└── index.ts              ← 套件定义，导出 {suiteName}Suite(runner) 函数
```

### eval.test.ts 标准模板

```typescript
import { getDefaultRunner } from "@deepagents/evals";
import { basicSuite } from "./index.js";

const runner = getDefaultRunner();
basicSuite(runner);
```

### index.ts 标准模板

```typescript
import { ls } from "langsmith/vitest";
import type { EvalRunner } from "@deepagents/evals";

export function basicSuite(runner: EvalRunner): void {
  ls.describe("basic", () => {
    ls.test(
      "test name",
      { inputs: { query: "..." } },
      async ({ inputs }) => {
        const result = await runner.run({ query: inputs.query });
        expect(result).toHaveFinalTextContaining("expected");
      }
    );
  });
}
```

## LangSmith 集成

### 自动上报机制

每个自定义断言器调用 `ls.logFeedback()` 上报结构化数据：

```typescript
// 断言成功/失败都会上报
expect(result).toHaveAgentSteps(3);
// → { key: "agent_steps", score: 3 }
// → { key: "expected_num_agent_steps", score: 3 }
// → { key: "match_num_agent_steps", score: 1 }  // 1=通过, 0=失败
```

### 手动上报

套件可额外上报自定义 feedback：

```typescript
ls.logFeedback({ key: "difficulty", score: 0.8 });
ls.logFeedback({ key: "case_name", value: "vague_request_1" });
```

### 轨迹上报

`DeepAgentEvalRunner.run()` 在每次调用后自动调用：

```typescript
ls.logOutputs({
  steps: trajectory.steps.length,
  finalText: getFinalText(trajectory),
  files: trajectory.files,
});
```
