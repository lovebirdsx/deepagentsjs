# 架构总览

## 系统定位

`evals/` 是 DeepAgents 框架的**行为评估层**，解决一个核心问题：

> "当使用真实 LLM 运行时，Agent 的行为是否符合预期？"

它不是单元测试（不 Mock LLM），而是端到端集成测试——每个测试用例都会真实调用 LLM API，观察 Agent 的行为轨迹，并断言结果满足预期。

## 核心概念

### 1. EvalRunner（评估执行器）

每个 Runner 封装了一个特定模型配置的 Agent。Runner 有两个核心操作：

- `run(params)` — 执行 Agent，返回 `AgentTrajectory`（行为轨迹）
- `extend(overrides)` — 派生出一个携带额外配置的 Runner

```typescript
// 获取默认 Runner（由 EVAL_RUNNER 环境变量决定）
const runner = getDefaultRunner();

// 用默认配置运行
const result = await runner.run({ query: "做某件事" });

// 带配置覆盖运行
const result = await runner
  .extend({ systemPrompt: "自定义提示词" })
  .run({ query: "做某件事", initialFiles: { "/data.txt": "内容" } });
```

### 2. AgentTrajectory（行为轨迹）

一次 Agent 运行的完整记录：

```typescript
interface AgentTrajectory {
  steps: AgentStep[];              // Agent 的每一步推理
  files: Record<string, string>;   // 最终文件系统状态
}

interface AgentStep {
  index: number;          // 从 1 开始的步骤序号
  action: AIMessage;      // LLM 的输出（文本 + 工具调用）
  observations: ToolMessage[];  // 工具执行结果
}
```

### 3. 自定义断言器

评估套件使用 4 个专为 Agent 行为定制的 Vitest 断言器：

| 断言器                              | 含义                     |
| ----------------------------------- | ------------------------ |
| `toHaveAgentSteps(n)`               | 轨迹恰好有 n 步          |
| `toHaveToolCallRequests(n)`         | 总工具调用次数为 n       |
| `toHaveToolCallInStep(step, match)` | 第 step 步有特定工具调用 |
| `toHaveFinalTextContaining(text)`   | 最终回复包含指定文本     |

### 4. LangSmith 集成

每个断言器在执行时自动向 LangSmith 上报 feedback：

```typescript
expect(result).toHaveAgentSteps(3);
// 自动上报: { agent_steps: 3, expected_num_agent_steps: 3, match_num_agent_steps: true }
```

## 执行流程

```
① 环境准备
   EVAL_RUNNER=sonnet-4-6
   LANGSMITH_API_KEY=...
   ANTHROPIC_API_KEY=...
        │
② Vitest 启动（每个套件目录）
   vitest.config.ts 配置:
     setupFiles: ["@deepagents/evals/setup"]
        │
③ Setup 阶段（setup.ts）
   registerDeepAgentRunner("sonnet-4-6", factory)
   registerDeepAgentRunner("gpt-4.1", factory)
   ... 共 7 个 Runner 注册到全局注册表
        │
④ 测试文件加载（eval.test.ts）
   import { basicSuite } from "./index.ts"
   const runner = getDefaultRunner()   ← 读取 EVAL_RUNNER
   basicSuite(runner)                  ← 注册 LangSmith 测试
        │
⑤ 逐个测试执行
   runner.extend({...}).run({ query, initialFiles })
        │
   DeepAgentEvalRunner.run():
   ├─ 将 initialFiles 字符串转为 LangGraph FileData
   ├─ 生成 UUID thread_id
   ├─ 调用 createDeepAgent().invoke()
   ├─ 解析输出为 AgentTrajectory
   └─ ls.logOutputs() 上报到 LangSmith
        │
⑥ 断言 + LangSmith feedback 上报
   expect(result).toHaveFinalTextContaining("...")
   ls.logFeedback({ key: "agent_steps", score: ... })
        │
⑦ 结果汇总
   LangSmith 实验页面可查看全部反馈
   支持跨模型、跨版本对比
```

## 套件分类

| 类别         | 套件                                             | 核心能力               |
| ------------ | ------------------------------------------------ | ---------------------- |
| **基础**     | basic, todos                                     | 提示遵从、工具选择决策 |
| **文件**     | files, summarization                             | 文件 I/O、大文件处理   |
| **记忆**     | memory, memory-agent-bench, memory-multiturn     | 长期记忆与上下文注入   |
| **技能**     | skills                                           | SKILL.md 动态加载      |
| **工具**     | tool-selection, tool-usage-relational, subagents | 工具路由与多步链式调用 |
| **高级**     | hitl, followup-quality                           | 人机协作、问题质量     |
| **外部基准** | external-benchmarks, tau2-airline, oolong        | 对标学术/工业基准      |
| **聚合**     | all                                              | 批量运行所有套件       |
