# 架构总览

## 这个包是什么

`deepagents-acp`（v0.1.8）是一个 **ACP 协议服务器**，它把 DeepAgents（基于 LangGraph 的 AI Agent 框架）包装成一个可以被 IDE 直接调用的标准化服务。

简单来说：IDE（如 Zed、JetBrains）通过 ACP 协议与这个服务器通信，服务器把 IDE 的消息转给 LangGraph Agent 处理，再把 Agent 的响应流式推送回 IDE。

## ACP 协议概述

ACP（Agent Client Protocol）是类似 LSP（Language Server Protocol）的开放标准，专为 IDE 与 AI Agent 之间的通信设计。

**核心特征：**
- 传输层：`stdio`（标准输入/输出），以 NDJSON 格式传输 JSON-RPC 2.0 消息
- 方向：IDE → Agent（请求），Agent → IDE（响应 + 通知）
- `stdout` 专用于协议通信，日志必须写 `stderr` 或文件

**与 LSP 的类比：**

| LSP                     | ACP               |
| ----------------------- | ----------------- |
| language server         | AI Agent 服务器   |
| textDocument/hover      | session/prompt    |
| textDocument/completion | session/new       |
| LSP client（IDE）       | ACP client（IDE） |

```
│  ┌─────────────────▼──────────────────────────┐    │
│  │   Message Adapter（adapter.ts）             │    │

│                    │                                │
│  ┌─────────────────▼──────────────────────────┐    │

└─────────────────────────────────────────────────────┘
```

## 核心组件职责

| 组件                   | 文件                        | 职责                                       |
| ---------------------- | --------------------------- | ------------------------------------------ |
| `DeepAgentsServer`     | `server.ts`                 | ACP 协议处理、会话管理、Agent 生命周期     |
| `adapter`              | `adapter.ts`                | ACP ↔ LangChain 消息格式双向转换           |
| `ACPFilesystemBackend` | `acp-filesystem-backend.ts` | 文件 I/O 代理（IDE 客户端 → 本地 FS 回退） |
| `Logger`               | `logger.ts`                 | 双通道日志（stderr + 持久文件）            |
| `cli`                  | `cli.ts`                    | 命令行启动入口，参数解析                   |
| `types`                | `types.ts`                  | 所有 TypeScript 类型定义                   |

## 数据流向一览

```
用户在 IDE 输入消息
    ↓ session/prompt（ACP 请求）
handlePrompt()
    ↓ 检查 slash command？
        是 → 直接返回响应（不走 LLM）
        否 ↓
acpPromptToHumanMessage()         # ACP ContentBlock[] → LangChain HumanMessage
    ↓
agent.stream()                    # LangGraph 流式推理
    ↓ 流式事件循环
        AIMessage → sendMessageChunk()   # 文本 → IDE 渲染对话
        AIMessage.tool_calls → sendToolCall()  # 工具调用通知
        ToolMessage → sendToolCallUpdate()     # 工具结果更新
        todos → sendPlanUpdate()               # 计划列表更新
    ↓
return { stopReason: "end_turn" }
    ↓ IDE 展示完整对话
```

## 设计原则

1. **stdout 保洁**：stdout 专属 ACP JSON-RPC，所有日志走 stderr 或文件
2. **降级回退**：ACP 文件操作失败时自动回退到本地文件系统
3. **流式优先**：Agent 响应以流的方式逐步推送，不等全量完成
4. **会话隔离**：每个 session 有独立的 threadId，LangGraph 按 threadId 做 checkpoint
5. **格式解耦**：adapter 层完全隔离协议格式与 Agent 内部格式
