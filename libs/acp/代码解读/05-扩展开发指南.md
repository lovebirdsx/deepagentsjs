## 一、如果你要构建一个 ACP Agent 服务器

### 最小实现

```typescript
import { AgentSideConnection, ndJsonStream } from "@agentclientprotocol/sdk"

// 1. 创建 stdio 传输层
const input = ReadableStream  // 包装 process.stdin
const output = WritableStream  // 包装 process.stdout
const stream = ndJsonStream(output, input)  // 注意：output 在前

// 2. 创建连接
const conn = new AgentSideConnection(
  (conn) => ({
    initialize: (params) => { /* 返回能力声明 */ },
    newSession: (params) => { /* 创建会话 */ },
    prompt: (params) => {
      // 核心处理：接收用户输入，返回 stopReason
      conn.sessionUpdate({ sessionUpdate: "agent_message_chunk", content: {...} })
      return { stopReason: "end_turn" }
    },
    cancel: (params) => {},
    authenticate: (params) => {},
    loadSession: (params) => {},
    setSessionMode: (params) => {},
  }),
  stream
)

await conn.closed  // 阻塞直到 IDE 断开
```

### 关键约束

1. **stdout 严禁写日志**：ACP 协议走 stdout，日志必须写 stderr 或文件
2. **protocolVersion 是 number**：不是字符串，否则 IDE 可能拒绝握手
3. **能力声明要准确**：声明了 `loadSession: true` 就必须实现 `loadSession`

## 二、消息格式转换层的设计

本项目的 `adapter.ts` 提供了一个清晰的模式：

### 原则

- **单向函数**：每个函数只做一个方向的转换
- **降级处理**：不认识的格式转成文本，不报错
- **优先级优化**：单纯文本直接返回 string，避免不必要的数组包装

### 你可能需要的转换对

| 场景 | 函数 |
|---|---|
| 用户输入 → LLM 输入 | `acpContentToLangChain()` |
| LLM 输出 → IDE 显示 | `langChainContentToACP()` |
| 提取工具调用 | `extractToolCalls()` |
| 生成工具标题 | `formatToolCallTitle()` |
| 确定工具种类 | `getToolCallKind()` |

## 三、会话状态设计模式

```typescript
// 核心会话状态结构（可参考）
interface Session {
  id: string             // 外部标识（给 ACP 用）
  threadId: string       // 内部标识（给 LLM 框架用）
  agentName: string      // 对应的 AI 配置
  mode: string           // 当前工作模式
  messages: Message[]    // 对话历史（用于回放）
  permissionCache: Map<string, Decision>  // 工具权限缓存
  createdAt: Date
  lastActivityAt: Date
}
```

**设计要点**：
- `id` 和 `threadId` 分离，可以在同一 session 内重置 LLM 历史（`/clear` 场景）
- `permissionCache` 跨请求生效，减少用户重复审批
- `lastActivityAt` 可用于实现过期清理

## 四、流式响应处理模式

LangGraph 流式事件的结构不稳定（不同版本、不同 Node 会有不同 key），需要防御性读取：

```typescript
for await (const event of agent.stream(...)) {
  let messages = []

  // 防御性提取：多条路径
  if (Array.isArray(event.messages))
    messages = event.messages
  else if (event.model_request?.messages)
    messages = event.model_request.messages
  else if (event.tools?.messages)
    messages = event.tools.messages

  for (const msg of messages) {
    if (isAIMessage(msg))   handleAI(msg)
    if (isToolMessage(msg)) handleTool(msg)
  }

  // 处理特殊更新
  if (event.todos) handlePlan(event.todos)
}
```

## 五、工具调用追踪模式

用 Map 追踪活跃工具调用，按 ID 关联结果：

```typescript
const activeToolCalls = new Map<string, ToolCallInfo>()

// AIMessage 触发
for (const tc of message.tool_calls) {
  const info = { id: tc.id, name: tc.name, args: tc.args, status: "pending" }
  notifyClient("tool_call", info)
  activeToolCalls.set(tc.id, info)
  info.status = "in_progress"
  notifyClient("tool_call_update", info)
}

// ToolMessage 完成
const info = activeToolCalls.get(toolMessage.tool_call_id)
info.status = "completed"
info.result = toolMessage.content
notifyClient("tool_call_update", info)
activeToolCalls.delete(toolMessage.tool_call_id)

// 取消时
for (const info of activeToolCalls.values()) {
  info.status = "cancelled"
  notifyClient("tool_call_update", info)
}
```

## 六、文件系统后端模式

代理模式（IDE → 本地 FS 降级）的标准实现：

```typescript
class ACPFilesystemBackend extends FilesystemBackend {
  async read(filePath) {
    if (!this.sessionId) return super.read(filePath)  // 无会话，本地读
    try {
      return await this.conn.readTextFile({ sessionId, path: resolve(filePath) })
    } catch {
      return super.read(filePath)  // 失败，本地读
    }
  }
}
```

**适用场景**：任何需要"优先从外部系统读取，失败降级本地"的操作。

## 七、日志系统设计

ACP 服务器的日志必须避开 stdout，参考 `logger.ts` 的双通道设计：

```typescript
class Logger {
  // stderr（调试时实时查看）
  // 文件（生产环境持久记录）

  private write(level, args) {
    if (this.debug || level >= "warn") {
      console.error(format(args))  // 写 stderr，不是 stdout！
    }
    if (this.fileStream) {
      this.fileStream.write(format(args) + "\n")
    }
  }
}
```

关闭时确保 flush：

```typescript
async close() {
  return new Promise(resolve => {
    this.fileStream.write(shutdownMarker, () => {
      this.fileStream.end(resolve)
    })
  })
}
```

## 八、Human-in-the-Loop 设计

权限审批的核心模式（三层决策）：

```typescript
async requestPermission(toolName, toolArgs) {
  // 第一层：永久缓存
  const cached = this.permissionCache.get(toolName)
  if (cached === "allow_always") return "allow"
  if (cached === "reject_always") return "reject"

  // 第二层：用户交互
  const result = await conn.requestPermission({
    options: ["allow-once", "allow-always", "reject-once", "reject-always"]
  })

  // 第三层：更新缓存
  if (result === "allow-always") {
    this.permissionCache.set(toolName, "allow_always")
  }

  return result.includes("allow") ? "allow" : "reject"
}
```

## 九、取消操作模式

AbortController 的标准接入：

```typescript
// 每次请求创建，确保不跨请求泄漏
this.abortController = new AbortController()
try {
  const stream = await agent.stream(input, { signal: abortController.signal })
  for await (const event of stream) {
    if (abortController.signal.aborted) {
      // 清理活跃工具调用
      return "cancelled"
    }
    // 处理事件
  }
} finally {
  this.abortController = null  // 清理引用
}

// 取消入口
cancel() {
  this.abortController?.abort()
}
```

## 十一、复制本项目的注意点

### 必须修改的地方

1. **依赖替换**：`deepagents` 和 `@agentclientprotocol/sdk` 是核心依赖，如果不用这两个包，需要重新实现协议层和 Agent 层
2. **checkpointer 外部注入**：当前 `MemorySaver` 是内部创建的，若需持久化，需在 `DeepAgentsServerOptions` 中添加 `checkpointer` 选项

### 可以直接复用的模式

- `adapter.ts` 中的所有工具函数
- `logger.ts` 的双通道日志设计
- `acp-filesystem-backend.ts` 的代理+降级模式
- `streamAgentResponse()` 中的事件处理循环
- AbortController 取消模式
- 权限缓存模式

### ACP SDK 版本兼容性

代码中存在多处兼容性处理：

```typescript
// readTextFile 返回字段兼容
const text = result.content ?? result.text ?? ""

// requestPermission 结果兼容
const outcome = result?.outcome as { outcome: string; optionId?: string }
```

选择 `@agentclientprotocol/sdk` 版本时，建议锁定小版本（`^0.18.0`），避免兼容性问题。
