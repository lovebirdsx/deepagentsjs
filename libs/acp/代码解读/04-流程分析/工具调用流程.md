# 工具调用流程

本文描述一次完整工具调用的全生命周期，以及 Human-in-the-Loop 权限审批机制。

---

## 工具调用总体流程

```
                    DeepAgentsServer                    LangGraph Agent
                          │                                    │
                          │<── agent.stream() ─────────────── │
                          │                                    │
                          │         AI 决定调用工具             │
                          │<── AIMessage {                     │
                          │     tool_calls: [{                 │
                          │       id: "call_abc123",           │
                          │       name: "read_file",           │
                          │       args: { path: "src/main.ts" }│
                          │     }]                             │
                          │   }                                │
                          │                                    │
              extractToolCalls(message)                        │
              → ToolCallInfo { id, name, args, status:"pending" }
                          │                                    │
         ┌────────────────┤                                    │
         │ sendToolCall() │  通知 IDE                          │
         │                │──> sessionUpdate(tool_call) ──────>│IDE
         │                │    { toolCallId, title, kind,      │
         │                │      status: "pending",             │
         │                │      locations: [{path, line}],    │
         │                │      input: args }                 │
         │                │                                    │
         │                │  更新状态                           │
         │ sendToolCall   │──> sessionUpdate(tool_call_update)>│IDE
         │ Update()       │    { toolCallId,                   │
         │                │      status: "in_progress" }       │
         └────────────────┤                                    │
                          │                                    │
                          │         工具实际执行（LangGraph 内部）│
                          │<── ToolMessage {                   │
                          │     tool_call_id: "call_abc123",   │
                          │     content: "文件内容..."          │
                          │   }                                │
                          │                                    │
              activeToolCalls.get(tool_call_id)                │
              toolCall.status = "completed"                    │
              toolCall.result = content                        │
                          │                                    │
         sendToolCallUpdate() ──> sessionUpdate(tool_call_update) ──> IDE
                          │    { toolCallId,                   │
                          │      status: "completed",          │
                          │      output: "文件内容...",         │
                          │      content: [{type:"content"...}]}
                          │                                    │
              activeToolCalls.delete(tool_call_id)             │
```

## ToolCallInfo 状态机

```
                  ┌─────────┐
                  │ pending │ ← extractToolCalls() 初始化
                  └────┬────┘
                       │ sendToolCallUpdate(in_progress)
                  ┌────▼──────────┐
                  │  in_progress  │
                  └────┬──────────┘
           ┌───────────┼───────────┐
           │           │           │
      ┌────▼────┐ ┌────▼──┐ ┌─────▼─────┐
      │completed│ │ error │ │ cancelled │
      └─────────┘ └───────┘ └───────────┘
```

`cancelled` 发生在：取消信号触发时，对所有 `activeToolCalls` 中尚未完成的工具调用统一标记。

## 工具种类映射（`getToolCallKind()`）

```typescript
const readTools    = ["read_file", "ls"]         → "read"
const searchTools  = ["grep", "glob"]            → "search"
const editTools    = ["write_file", "edit_file"] → "edit"
const executeTools = ["execute", "shell", "terminal"] → "execute"
const thinkTools   = ["write_todos"]             → "think"
// 其他自定义工具                                 → "other"
```

IDE 根据 `kind` 展示不同图标和颜色。

## ACP 通知格式详解

### tool_call（初始通知）

```json
{
  "sessionId": "sess_abc123",
  "update": {
    "sessionUpdate": "tool_call",
    "toolCallId": "call_xyz789",
    "title": "Reading src/main.ts",
    "kind": "read",
    "status": "pending",
    "locations": [{ "path": "/workspace/src/main.ts", "line": 42 }],
    "input": { "path": "src/main.ts" }
  }
}
```

### tool_call_update（状态更新）

```json
// 进行中
{
  "sessionUpdate": "tool_call_update",
  "toolCallId": "call_xyz789",
  "status": "in_progress"
}

// 完成（带结果）
{
  "sessionUpdate": "tool_call_update",
  "toolCallId": "call_xyz789",
  "status": "completed",
  "output": "文件内容...",
  "content": [{
    "type": "content",
    "content": { "type": "text", "text": "文件内容..." }
  }]
}
```

## Human-in-the-Loop 权限审批

当 Agent 配置了 `interruptOn`，LangGraph 会在工具执行前暂停，服务器调用 `requestToolPermission()`：

### 触发条件

```typescript
// 配置示例
const agent = new DeepAgentsServer({
  agents: {
    interruptOn: {
      write_file: true,       // 始终需要审批
      edit_file: true,        // 始终需要审批
      execute: {              // 有选项的审批
        allowedDecisions: ["approve", "edit", "reject"]
      }
    }
  }
})
```

### 权限检查流程

```typescript
async requestToolPermission(session, conn, toolCall) {
  // 1. 检查会话级缓存
  const cached = session.permissionDecisions.get(toolCall.name)
  if (cached === "allow_always") return "allow"    // 快速通过
  if (cached === "reject_always") return "reject"  // 快速拒绝

  // 2. 向 IDE 请求用户决策
  const result = await conn.requestPermission({
    sessionId,
    toolCall: {
      toolCallId, title, kind, status: "pending", input: args
    },
    options: [
      { optionId: "allow-once",    kind: "allow_once" },
      { optionId: "allow-always",  kind: "allow_always" },
      { optionId: "reject-once",   kind: "reject_once" },
      { optionId: "reject-always", kind: "reject_always" },
    ]
  })

  // 3. 处理用户选择
  switch (result.outcome.optionId) {
    case "allow-always":
      session.permissionDecisions.set(toolCall.name, "allow_always")
      return "allow"
    case "reject-always":
      session.permissionDecisions.set(toolCall.name, "reject_always")
      return "reject"
    case "reject-once":
      return "reject"
    default:  // allow-once 或 cancelled
      return "allow"
  }
}
```

### 权限缓存

`permissionDecisions` 是 `Map<toolName, "allow_always" | "reject_always">`，存在 `SessionState` 中：
- 跨请求生效（同一 session 内）
- `/clear` 后不重置（sessionId 不变）
- 服务器重启后丢失

## 终端执行（`executeWithTerminal()`）

当客户端支持 `terminal` 能力时，`execute`/`shell` 类工具可通过 IDE 终端执行（提供实时流式输出）：

```typescript
async executeWithTerminal(session, conn, toolCall) {
  const terminal = await conn.createTerminal({
    sessionId: session.id,
    command: "/bin/bash",
    args: ["-c", toolCall.args.command],
    cwd: this.workspaceRoot,
  })

  await sendToolCallUpdate({ status: "in_progress" })

  const exitResult = await terminal.waitForExit()
  const outputResult = await terminal.currentOutput()
  await terminal.release()

  return { output: outputResult.output, exitCode: exitResult.exitCode }
}
```

失败时静默回退：捕获异常返回错误信息，不中断流程。

## 并发工具调用

LangGraph 支持并发工具调用（一次 AIMessage 携带多个 `tool_calls`）。服务器通过 `activeToolCalls` Map 追踪所有活跃调用：

```typescript
// 批量注册
for (const toolCall of toolCalls) {
  await sendToolCall(toolCall)
  activeToolCalls.set(toolCall.id, toolCall)
  toolCall.status = "in_progress"
  await sendToolCallUpdate(toolCall)
}

// 按 tool_call_id 匹配结果
// ToolMessage.tool_call_id → activeToolCalls.get(id)
```

每个 ToolMessage 携带 `tool_call_id` 字段，确保结果与调用一一对应。
