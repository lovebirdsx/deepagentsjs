# 会话持久化与恢复

本文描述基于 LangGraph `MemorySaver` 的会话持久化机制，以及加载历史会话时的回放流程。

---

## 持久化机制

### LangGraph Checkpointer

```typescript
// server.ts 构造函数
this.checkpointer = new MemorySaver()

// 创建 Agent 时传入共享 checkpointer
const agent = createDeepAgent({
  ...
  checkpointer: this.checkpointer,  // 共享，多 Agent 使用同一个
})
```

`MemorySaver` 是 LangGraph 的内存检查点实现，按 `thread_id` 存储状态快照。

### thread_id 与 sessionId 的关系

```
sessionId = "sess_abc123def456"    ← ACP 层标识（不变）
threadId  = "xxxxxxxx-xxxx-..."    ← LangGraph 层标识（可重置）
```

- **一个 session 对应一个 threadId**
- 执行 `/clear` 时，`session.id` 不变，`session.threadId` 重新生成
- 新的 `threadId` → LangGraph 从空状态开始，等效新会话

### 状态存储位置

```typescript
// agent.stream() 调用时
const config = {
  configurable: { thread_id: session.threadId },
  signal: ...,
}
const stream = await agent.stream({ messages: [humanMessage] }, config)
```

每次流式调用结束后，LangGraph 自动将完整对话状态（messages、todos 等）以 `thread_id` 为 key 写入 `MemorySaver`。

### 存储的数据结构

```typescript
// MemorySaver 内部存储的 checkpoint 结构
{
  checkpoint: {
    channel_values: {
      messages: BaseMessage[],  // 完整对话历史
      // 其他 LangGraph 节点状态
    }
  }
}
```

## 读取检查点：`handleLoadSession()`

当 IDE 切换窗口或重新打开 panel 时，触发 `session/load`：

```typescript
async handleLoadSession(params, conn) {
  const session = sessions.get(params.sessionId)
  if (!session) throw new Error("Session not found")

  session.lastActivityAt = new Date()

  // 重新关联 ACP Backend（如果有）
  acpBackend.setSessionId(sessionId)

  // 回放历史消息
  await this.replaySessionHistory(session, conn)

  // 重新发送可用命令
  conn.sessionUpdate({ available_commands_update, allCommands })

  return { modes: { availableModes, currentModeId: session.mode ?? "agent" } }
}
```

## 历史回放：`replaySessionHistory()`

```typescript
async replaySessionHistory(session, conn) {
  // 1. 从 LangGraph checkpointer 获取历史
  let messages: BaseMessage[] = []
  try {
    const checkpoint = await checkpointer.getTuple({
      configurable: { thread_id: session.threadId }
    })
    messages = checkpoint?.checkpoint?.channel_values?.messages ?? []
  } catch {
    // checkpointer 无数据时不报错
  }

  // 2. 降级：从内存 session.messages 获取
  if (messages.length === 0) {
    messages = session.messages as BaseMessage[]
  }

  // 3. 逐条重播到 IDE
  for (const message of messages) {
    if (HumanMessage.isInstance(message)) {
      // 用户消息
      await sendMessageChunk(session.id, conn, "user", langChainMessageToACP(message))
    }

    if (AIMessage.isInstance(message)) {
      // AI 文本响应
      if (message.content) {
        await sendMessageChunk(session.id, conn, "agent", langChainMessageToACP(message))
      }
      // AI 工具调用（标记为 completed 状态）
      const toolCalls = extractToolCalls(message)
      for (const toolCall of toolCalls) {
        await sendToolCall(session.id, conn, toolCall)
        await sendToolCallUpdate(session.id, conn, { ...toolCall, status: "completed" })
      }
    }

    // ToolMessage 不单独重播（工具结果已通过 tool_call_update 覆盖）
  }
}
```

## 回放时序图

```
IDE 重新打开会话面板                          DeepAgentsServer
         │                                         │
         │── session/load({ sessionId }) ─────────>│
         │                                         │
         │                          checkpointer.getTuple({ thread_id })
         │                          → 获取历史 messages
         │                                         │
         │         ─── 逐条重播 ────               │
         │<── sessionUpdate(user_message_chunk)    │  "帮我分析这个文件"
         │<── sessionUpdate(agent_message_chunk)   │  "好的，我来分析..."
         │<── sessionUpdate(tool_call)             │  read_file("src/main.ts")
         │<── sessionUpdate(tool_call_update)      │  status: "completed"
         │<── sessionUpdate(agent_message_chunk)   │  "分析结果..."
         │                                         │
         │<── sessionUpdate(available_commands)    │
         │<── { modes: { currentModeId: "agent" } }│
```

IDE 收到所有回放通知后，能够完整重建对话界面。

## 持久化的局限性

### 当前实现（MemorySaver）

| 特性 | 状态 |
|---|---|
| 跨请求持久 | ✅ 同一进程内 |
| 服务器重启后持久 | ❌ 内存存储，重启丢失 |
| 多进程共享 | ❌ 进程隔离 |

### 升级为持久存储

替换 `MemorySaver` 为任何 LangGraph 兼容的 checkpointer：

```typescript
// PostgreSQL 持久化示例
import { PostgresSaver } from "@langchain/langgraph-checkpoint-postgres"

const checkpointer = await PostgresSaver.fromConnString(
  "postgresql://user:pass@localhost/db"
)

new DeepAgentsServer({
  agents: {
    name: "my-agent",
    // checkpointer 会在 createDeepAgent() 中使用
  }
})
```

> 当前代码中 `checkpointer` 是在 `DeepAgentsServer` 内部创建的，无法从外部注入。若需持久化，需修改 `DeepAgentsServerOptions` 添加 `checkpointer` 字段，或 fork 修改。

## 数据保留周期

`MemorySaver` 不会自动清理旧数据，数据在进程生命周期内一直存在。如果对话量大，需注意内存增长。

`/clear` 命令虽然重置了 `threadId`，旧的 checkpoint 数据仍保留在 `MemorySaver` 中（只是不再被访问）。
