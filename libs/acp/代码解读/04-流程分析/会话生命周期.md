# 会话生命周期

本文描述从创建会话到完成一次对话交互的完整流程。

---

## 总体时序

```
IDE                                  DeepAgentsServer                    DeepAgent (LangGraph)
 │                                          │                                    │
 │── session/new ────────────────────────>  │                                    │
 │   { configOptions: { agent: "xxx" } }    │                                    │
 │                           generateSessionId()                                 │
 │                           generateThreadId()                                  │
 │                           createAgent() → createDeepAgent()                   │
 │<── { sessionId, modes } ───────────────  │                                    │
 │                                          │                                    │
 │<── sessionUpdate (available_commands) ── │ (异步发送)                          │
 │                                          │                                    │
 ├─────────── 用户输入阶段 ──────────────────┤                                    │
 │                                          │                                    │
 │── session/prompt ─────────────────────> │                                    │
 │   { sessionId, prompt: [...] }           │                                    │
 │                           检查 slash command?                                  │

```

## 创建会话：`handleNewSession()`

```typescript
// 1. 生成标识符
const sessionId = generateSessionId()   // "sess_" + 16 hex chars
const threadId = crypto.randomUUID()    // LangGraph 检查点 key

// 2. Agent 选择
const agentName = configOptions?.agent  // 客户端指定
                ?? [...agentConfigs.keys()][0]  // 默认第一个

// 3. 创建 SessionState
const session: SessionState = {
  id: sessionId,
  agentName,
  threadId,
  messages: [],
  createdAt: new Date(),
  lastActivityAt: new Date(),
  mode: params.mode,  // 可在创建时指定初始模式
}
sessions.set(sessionId, session)

// 4. 懒加载 Agent（首次使用时才创建）
if (!agents.has(agentName)) {
  this.createAgent(agentName)
}

// 5. 异步发送可用命令（不阻塞响应）
conn.sessionUpdate({ available_commands_update, availableCommands })
  .catch(err => log("Failed to send commands:", err))

return { sessionId, modes: { availableModes, currentModeId } }
```

**设计要点**：命令更新是异步发送的（`.catch()` 处理），不阻塞 `session/new` 的响应，客户端更快得到 sessionId。

## 处理用户输入：`handlePrompt()`

### Slash Command 拦截

在发给 AI 之前先检查：

```typescript
const text = prompt.text.trim()
if (text.startsWith("/")) {
  const [command] = text.split(/\s+/, 1)
  switch (command.slice(1).toLowerCase()) {
    case "plan":
    case "agent":
    case "ask":
      session.mode = commandName
      await sendMessageChunk("Switched to X mode.")
      return { stopReason: "end_turn" }  // 直接返回，不走 AI

    case "clear":
      session.messages = []
      session.threadId = crypto.randomUUID()  // 新的 thread → 新的历史
      await sendMessageChunk("Conversation cleared. Starting fresh.")
      return { stopReason: "end_turn" }

    case "status":
      await sendMessageChunk(`**Agent:** ${name}\n**Mode:** ${mode}\n...`)
      return { stopReason: "end_turn" }
  }
}
```

### Agent 流式处理

```typescript
const humanMessage = acpPromptToHumanMessage(prompt)
session.messages.push(humanMessage)

const stream = await agent.stream(
  { messages: [humanMessage] },
  { configurable: { thread_id: session.threadId }, signal: abortController.signal }
)

for await (const event of stream) {
  // ... 处理事件（见工具调用流程文档）
}
return "end_turn"
```

## AbortController 模式

每次 `handlePrompt()` 创建一个新的 `AbortController`：

```typescript
// handlePrompt()
this.currentPromptAbortController = new AbortController()
try {
  // ... 流式处理
} finally {
  this.currentPromptAbortController = null  // 清理
}

// handleCancel()
this.currentPromptAbortController?.abort()
```

`signal` 传给 `agent.stream()`，LangGraph 在下一个可取消点终止流。

## 模式系统

三种模式（当前为语义标记，实际约束由 Agent 系统提示实现）：

| 模式 ID | 名称 | 语义 |
|---|---|---|
| `agent` | Agent Mode | 完全自主，可读写文件、执行命令 |
| `plan` | Plan Mode | 规划讨论，只读 |
| `ask` | Ask Mode | 问答，不做文件修改 |

切换方式：
- slash 命令：`/plan`、`/agent`、`/ask`
- ACP 请求：`session/set_mode({ modeId })`
- 初始化时：`session/new({ mode })` 参数

---

## 会话生命结束

当前实现没有显式的 `session/end` 方法。会话在以下情况结束：
- 服务器进程退出（`SIGINT`/`SIGTERM`）
- `AgentSideConnection.closed` Promise resolve（stdio 关闭）
- 调用 `server.stop()`（`sessions.clear()`）

所有会话状态保存在内存，服务器重启后丢失（LangGraph checkpoint 除外）。
