# 数据结构与类型定义

**文件路径：** `src/types.ts`

所有核心 TypeScript 类型定义集中在此文件，以下逐一解析。

---

## DeepAgentConfig

Agent 配置，继承自 `deepagents` 包的 `CreateDeepAgentParams`，并新增 ACP 特有字段。

```typescript
interface DeepAgentConfig extends CreateDeepAgentParams {
  name: string           // 必填，唯一标识符，用于 session 路由
  description?: string   // 人类可读描述，ACP 列出 agent 时展示

  // 自定义 slash 命令（与内置命令合并后发送给客户端）
  commands?: Array<{
    name: string
    description: string
    input?: { hint: string }  // 命令参数提示（如 "environment (staging/production)"）
  }>
}
```

**继承的关键字段**（来自 `CreateDeepAgentParams`）：

| 字段             | 类型                                | 说明                   |
| ---------------- | ----------------------------------- | ---------------------- |
| `model`          | `string`                            | LLM 模型名称           |
| `tools`          | `StructuredTool[]`                  | 自定义工具列表         |
| `systemPrompt`   | `string`                            | 自定义系统提示词       |
| `middleware`     | `AgentMiddleware[]`                 | 中间件管道             |
| `subagents`      | `array`                             | 可委托的子 Agent       |
| `responseFormat` | `any`                               | 结构化输出 schema      |
| `contextSchema`  | `InteropZodObject`                  | 输入校验 schema        |
| `interruptOn`    | `Record<string, InterruptConfig>`   | HITL 工具审批配置      |
| `store`          | `any`                               | 自定义存储后端         |
| `backend`        | `BackendProtocol \| BackendFactory` | 文件系统后端           |
| `skills`         | `string[]`                          | SKILL.md 文件路径列表  |
| `memory`         | `string[]`                          | AGENTS.md 文件路径列表 |

## SessionState

一个活跃对话会话的内存状态：

```typescript
interface SessionState {
  id: string              // 会话 ID（格式：sess_xxxxxxxxxxxxxxxx）
  agentName: string       // 对应的 Agent 名称
  threadId: string        // LangGraph thread_id（UUID 格式，用于 checkpoint）

  messages: unknown[]     // 对话历史（BaseMessage 数组）
  createdAt: Date
  lastActivityAt: Date

  mode?: string           // 当前模式："agent" | "plan" | "ask"

  // 工具权限缓存（"Always allow" / "Always reject" 的持久化）
  permissionDecisions?: Map<string, "allow_always" | "reject_always">
}
```

**关键点**：`threadId` 和 `id` 是两个不同的标识符：
- `id`（sessionId）：ACP 层使用，标识一次会话连接
- `threadId`：LangGraph 层使用，通过 checkpointer 存储对话状态

执行 `/clear` 命令时，`id` 不变但 `threadId` 重置，等效于在同一 ACP 会话中开启全新对话。

## ToolCallInfo

单次工具调用的追踪状态：

```typescript
interface ToolCallInfo {
  id: string           // 工具调用唯一 ID（格式：call_xxxxxxxxxxxx）
  name: string         // 工具名（如 "read_file", "execute"）
  args: Record<string, unknown>  // 工具参数

  status: "pending" | "in_progress" | "completed" | "failed" | "cancelled" | "error"

  result?: unknown     // 工具执行结果（completed 或 error 状态时有值）
  error?: string       // 错误信息（failed 状态时有值）
}
```

**状态流转**：

```
pending → in_progress → completed
                     ↘ failed
                     ↘ error
                     ↘ cancelled（流被中止时）
```

## PlanEntry

`write_todos` 工具输出的计划条目，对应 ACP 的 plan update：

```typescript
interface PlanEntry {
  content: string     // 任务描述文本
  priority?: "high" | "medium" | "low"
  status: "pending" | "in_progress" | "completed" | "skipped"
  // 注意：LangGraph 中的 "cancelled" 状态被映射为 "skipped"
}
```

## StopReason

Agent 响应结束的原因（作为 `session/prompt` 的最终返回值）：

```typescript
type StopReason =
  | "end_turn"           // 正常完成
  | "max_tokens"         // token 上限
  | "max_turn_requests"  // 轮次请求上限
  | "refusal"            // 模型拒绝响应
  | "cancelled"          // 用户取消（session/cancel）
```

## ACPCapabilities

存储从客户端 `initialize` 请求中解析出的能力标记：

```typescript
interface ACPCapabilities {
  fsReadTextFile?: boolean   // 客户端支持提供文件内容（含未保存缓冲区）
  fsWriteTextFile?: boolean  // 客户端支持接收文件写入
  terminal?: boolean         // 客户端支持内嵌终端
  loadSession?: boolean      // 客户端支持会话加载
  modes?: boolean            // 客户端支持模式切换
  commands?: boolean         // 客户端支持 slash 命令
}
```

## 认证方法类型

三种 ACP 认证方式（服务器在 `initialize` 响应中声明，帮助 IDE 引导用户配置凭据）：

### ACPAuthMethodAgent（agent 型）
```typescript
interface ACPAuthMethodAgent {
  id: string
  name: string
  type?: "agent"       // 可省略，默认就是 agent 型
  description?: string // 说明文字
}
```

### ACPAuthMethodEnvVar（环境变量型）
```typescript
interface ACPAuthMethodEnvVar {
  id: string
  name: string
  type: "env_var"
  link?: string          // 获取凭据的外部链接
  vars: ACPAuthEnvVar[]  // 需要设置的环境变量列表
}

interface ACPAuthEnvVar {
  name: string       // 环境变量名（如 "ANTHROPIC_API_KEY"）
  label?: string     // IDE 展示的友好名称
  secret?: boolean   // 是否为密码型输入（默认 true）
  optional?: boolean // 是否可选（默认 false）
}
```

### ACPAuthMethodTerminal（交互终端型）
```typescript
interface ACPAuthMethodTerminal {
  id: string
  name: string
  type: "terminal"
  args?: string[]             // 登录时传给 Agent 二进制的额外参数
  env?: Record<string, string> // 登录时的额外环境变量
}
```

## ServerEvents

服务器事件接口（供外部监听，目前未完全实现事件系统）：

```typescript
interface ServerEvents {
  sessionCreated: (session: SessionState) => void
  sessionEnded: (sessionId: string) => void
  promptStarted: (sessionId: string, prompt: string) => void
  promptCompleted: (sessionId: string, stopReason: StopReason) => void
  toolCallStarted: (sessionId: string, toolCall: ToolCallInfo) => void
  toolCallCompleted: (sessionId: string, toolCall: ToolCallInfo) => void
  error: (error: Error) => void
}
```
